Введение

Актуальность темы исследования

Современные системы поддержки принятия решения и автоматизированные системы управление построены на последних достижениях прикладных наук. Одной из таких наук является математическое программирование [ REF _Ref3142837 \r \h 1, REF _Ref5268274 \r \h 30, REF _Ref21960389 \r \h 32, REF _Ref21960421 \r \h 33, REF _Ref21960522 \r \h 44]. С помощью математического программирования решаются разнообразные задачи, так или иначе являющиеся составными частями автоматизированных систем управления (АСУ), систем автоматизированного проектирования (САПР) и систем поддержки принятия решения (СППР).

Важным подклассом задач математического программирования являются линейные задачи целочисленного программирования [ REF _Ref5265035 \r \h 12, REF _Ref21961045 \r \h 14, REF _Ref21960340 \r \h 15, REF _Ref5264975 \r \h 28, REF _Ref21961062 \r \h 34, REF _Ref21961978 \r \h 36, REF _Ref5267907 \r \h 46, REF _Ref21961120 \r \h 53, REF _Ref5266628 \r \h 61, REF _Ref5268174 \r \h 62]. С помощью них формализуются многие реальные задачи оптимального планирования [ REF _Ref21961418 \r \h 6, REF _Ref21961361 \r \h 7, REF _Ref21961287 \r \h 8, REF _Ref5266856 \r \h 13, REF _Ref21961310 \r \h 19, REF _Ref5266169 \r \h 24, REF _Ref21961335 \r \h 29, REF _Ref5267617 \r \h 31, REF _Ref5266980 \r \h 35, REF _Ref5265928 \r \h 52,  REF _Ref22930275 \r \h 58, REF _Ref5266942 \r \h 59] и, в частности, такие типовые постановки задач этого класса, как: основная производственная задача с дискретным характером производимой продукции, задача о ранце, задача о загрузке транспорта и многие другие.

Необходимо отметить, что многие задачи нелинейного программирования могут быть сведены к линейным задачам.

В настоящее время существует ряд оптимизационных методов, позволяющих решать задачи этого класса, такие как: методы неявного перебора, метод правильных отсечений, метод ветвей и границ. Каждый из этих методов имеет свои серьезные недостатки, ограничивающие их применение к решению высокоразмерных оптимизационных задач, что делает актуальной разработку более эффективного метода и соответствующего алгоритма решения линейных целочисленных оптимизационных задач: более надежного и обладающего меньшей вычислительной сложностью [ REF _Ref21961893 \r \h 11, REF _Ref5267970 \r \h 39, REF _Ref5268314 \r \h 40, REF _Ref21961755 \r \h 45, REF _Ref5267574 \r \h 50, REF _Ref5264900 \r \h 55].

Цель работы

Целью работы является разработка комбинированного метода для решения линейных задач целочисленного программирования: обоснование, разработка алгоритмической и программной реализаций, исследование его свойств, и решение с помощью него задачи по нахождению оптимальной топологии ЛВС.

Задачи исследования:

решить с помощью разработанного комбинированного метода задачу по нахождению оптимальной топологии ЛВС.

обосновать идеи и концепцию разработанного комбинированного метода решения линейных задач целочисленного программирования;

разработать алгоритмическую реализацию комбинированного метода и эффективную программную реализацию комбинированного метода;

исследовать свойства комбинированного метода и сравнить его с существующими методами, которые решают задачи этого же класса.

Методика исследования

В работе применяются методы системного анализа, в том числе математического программирования, машинного эксперимента и объектно-ориентированного проектирования программ.

Результаты, выносимые на защиту:

результаты решения задачи по нахождению оптимальной топологии ЛВС при помощи комбинированного метода;

комбинированный метод для решения задач линейного целочисленного программирования;

результаты исследования свойств комбинированного метода и его результаты сравнения с существующими методами.

Научная новизна

впервые предложена концепция комбинированного метода решения линейных задач целочисленного программирования, объединяющая методы линейного программирования, неявного перебора по векторной решетке и эвристические подходы для эффективного решения задач большой размерности. Использование эвристических подходов в комбинированном методе не влияет на возможность нахождения строго оптимального решения, а лишь обеспечивает более быстрый его поиск;

проведено всестороннее сравнение разработанного метода с различными известными универсальными методами решения линейных задач целочисленного программирования (в том числе и с использованием машинного эксперимента), показавшее его преимущества;

поставлены и решены задачи оптимизации топологии ЛВС с использованием разработанного метода.

Практическая значимость

В результате выполнения диссертационной работы автором получены следующие результаты:

с использованием объектно-ориентированного подхода осуществлена программная реализация комбинированного метода, позволяющая подстраивать алгоритм метода под конкретную задачу с помощью использования разнообразных эвристик, и с помощью этой программной реализации получено решение задачи определения оптимальной топологии локальной вычислительной сети;

разработана эффективная библиотека современных оптимизационных методов для решения линейных задач целочисленного программирования;

результаты, полученные в работе, внедрены на предприятии РКК «Энергия». С помощью разработанного программного обеспечения была решена задача по определению оптимальной топологии ЛВС этого предприятия. Кроме того, осуществлено внедрение разработанного программного обеспечения в учебный процесс МАИ в составе лабораторного комплекса по изучению методов решения задач линейного целочисленного программирования.

Разработанное программное обеспечение зарегистрировано в Реестре программ для ЭВМ [ REF _Ref26274008 \r \h 21].

Апробация работы

Идеи и концепции, предложенные в диссертационной работе, докладывались и обсуждались на различных конференциях, в том числе международных, в частности, на 5, 6, 7, 8 и 11 международных научно-технических семинарах «Современные технологии в задачах управления автоматики и обработки информации» (Россия, Алушта, 1996, 1997, 1998, 1999, 2002), на восьмой международной научно-технической конференции студентов и аспирантов «Радиоэлектроника, электротехника и энергетика» (Москва, МЭИ, 2002).

Разработка комбинированного метода осуществлена в рамках плановой НИР ПН 235 (302-98-05) кафедры 302 МАИ, выполненной на основе конкурса грантов Министерства образования в 1998-1999 г.

Результаты работы отражены в 11 публикациях.

Основное содержание работы

Во введении обоснована актуальность темы диссертации, сформулирована цель, задачи и основные положения, выносимые на защиту, научная новизна и практическая значимость диссертационной работы.

В первой главе производится постановка задачи, решению которой посвящена эта диссертация, а именно формулированию задачи по определению оптимальной топологии ЛВС, содержательная постановка которой взята с предприятия РКК «Энергия», перед инженерами которой возникли потребности в решении некоторых оптимизационных задач, а именно: нахождение оптимальной структуры для локальной вычислительной сети, ее формализации и анализу текущего состояния достижений математического программирования в направлении линейных целочисленных оптимизационных задач.

Во второй главе диссертации производится обоснование комбинированного метода для решения задач линейного целочисленного программирования. В этой главе приведена математическая база комбинированного метода. Кроме того, вторая глава содержит описание алгоритма комбинированного метода с детализацией различных вариантов его функционирования. Особенности всех прочих оптимизационных методов, используемых в комбинированном методе, и их модификации также приведены во второй главе.

Третья глава посвящена анализу свойств комбинированного метода. Для этого производится разбор ряда тестовых задач, которые были решены классическими методам или вручную (несложные задачи малой размерности). Целочисленные линейные задачи являются важным подклассом задач дискретного программирования. Несвязность области допустимых решений этого класса задач порождает дополнительные по сравнению с задачами линейного и нелинейного программирования [ REF _Ref21962471 \r \h 5, REF _Ref21962491 \r \h 41, REF _Ref21962415 \r \h 56, REF _Ref5266351 \r \h 57, REF _Ref21962427 \r \h 60] проблемы разработки методов их решения: невозможность в полной мере использования итеративных процедур непрерывного движения в пространстве оптимизационных переменных; неправомерность в общем случае применения подхода, основанного на округлении наилучшего нецелочисленного решения и др. Указанные особенности линейных задач целочисленного программирования приводят к тому, что алгоритмическая и вычислительная сложность методов их решения является высокой.

При их решении могут использоваться методы решения линейных целочисленных задач как составная часть процесса поиска оптимального решения. Одним из методов решения задач частично-целочисленного программирования является метод декомпозиции. В этом методе происходит разделение задачи на две подзадачи: задачу линейного программирования и целочисленную задачу, которую после этого приходится многократно решать, поэтому и в частично-целочисленном программировании наличие эффективных методов решения целочисленных задач имеет большое значение.

Правильным оно называется потому, что формируется таким образом, что отсекает от допустимой области предыдущей задачи ее оптимальное решение, не отсекая в то же время ни одного допустимого решения исходной задачи. В этих случаях на основе свойств правильности введенных ограничений сохраняют справедливость все рассуждения, приведенные в п. 2 и 4: оптимальное решение расширенной задачи линейного программирования, принадлежащее  EMBED Equation.3  , является оптимальным решением исходной задачи. Метод ветвей и границ

Этот метод является наиболее общим среди всех методов дискретного программирования и не имеет принципиальных ограничений по применению. Его сущности заключается в направлении ветвления [ REF _Ref21962105 \r \h 27] области допустимых решений оптимизационной задачи с целью локализации оптимального решения в одной области из ее подобластей. При применении метода ветвей и границ к каждой конкретной задаче в первую очередь должны быть определены две важнейшие его процедуры:

ветвление множества возможных решений;

вычисление нижних и верхних оценок целевой функции

Основным недостатком метода ветвей и границ, затрудняющим его программную реализацию, является отсутствие данных о максимальном количестве элементов множества номеров подмножеств  EMBED Equation.3   (или  EMBED Equation.3  ), находящихся на концах ветвей и ветвление которых еще не прекращено, что в ряде случаев может приводить к переполнению памяти компьютера.

Использование эвристических алгоритмов для улучшения свойств оптимизационных методов решения задач линейного программирования

Иногда при решении оптимизационных задач имеет смысл использовать некоторую априорную информацию о решаемой задаче, если она есть, и если это позволить ускорить процедуру нахождения оптимального решения. В этом случае появляются так называемые эвристические алгоритмы, и существующие оптимизационные могут получать некоторые модификации, связанные с использованием эвристик.

Применение эвристических алгоритмов может приводить к тому, что оптимизационный метод теряет свою универсальность, что не является положительным моментом. При решении задачи по определению оптимальной топологии ЛВС будет использована эвристика, которая позволит значительно улучшить свойства комбинированного метода. Для понимания сути применения эвристических алгоритмов в решении оптимизационных задач, необходимо рассмотреть некоторые примеры.

Метод минимального элемента для поиска допустимого решения транспортной задачи

Одним из примеров использования эвристики в решении оптимизационных задач может являться специальный алгоритм определения допустимого решения транспортной задачи. Несмотря на то, что транспортная задача относится к более широкому классу задач линейного программирования, чем целочисленные задачи, ее рассмотрение с точки зрения применения эвристик является хорошим примером.

В этом алгоритме используются некоторые идеи, например:

способ заполнения матрицы при процедуре пересчета, когда берется минимальное значение двух незаполненных перевозок;

тот факт, что порядок заполнения элементов в матрице жестко не задан,

которые не применимы для всех задач этого класса, а применяются лишь при решении транспортной задачи, так как используют конкретные свойства именно этой задачи. Но использование этого алгоритма в решении транспортной задачи гораздо выгоднее, чем применение общего подхода для поиска допустимого решения, так значительно быстрее и проще дает требуемый результат.

Предпосылки создания нового метода для решения линейных целочисленных задач

Таким образом, из предыдущего обсуждения выявлено, что каждый из рассмотренных методов решения задач данного класса имеет свои недостатки, ограничивающие их применение к решению высокоразмерных оптимизационных задач [ REF _Ref5264900 \r \h 55]. В методах неявного перебора вычислительная сложность растет экспоненциально с ростом размерности оптимизационной задачи по числу переменных. Метод правильных отсечений (алгоритм Гомори) имеет целый спектр недостатков. Это и отсутствие хотя бы одного найденного допустимого решения до конца работы алгоритма, и накопление ошибок вычислений при итерациях симплекс-метода и полная неясность с числом больших итераций (количеством вводимых в задачу правильных отсечений, причем, как правило, очень большим даже в задачах сравнительно небольшой размерности). Последние два недостатка делают работу алгоритма ненадежной. Ненадежность и вычислительная сложность алгоритма резко возрастают с увеличением размерности задачи как по числу переменных, так и по числу ограничений. Метод ветвей и границ применительно к указанному классу задач порождает проблему хранения данных по итогам решения задач линейного программирования на концах ветвей: чтобы исключить повторное решение с самого начала задач линейного программирования, получающихся в результате ветвления, и применить двойственный вариант метода, необходимо для каждой задачи хранить полную структуру исходных данных и заключительную симплекс-таблицу. Отсутствие каких-либо оценок по количеству ветвлений до полного решения задачи и порождает указанную проблему. Кроме того, последовательное использование симплекс-метода после каждого ветвления при большом числе ветвлений приводит, как и в случае алгоритма Гомори, к накоплению ошибок вычислений с теми же самыми последствиями. Альтернативой такому подходу в методах правильных отсечений и ветвей и границ является решение задач линейного программирования на отдельных итерациях методов с самого начала, но это с очевидностью приводит к еще более резкому возрастанию вычислительной сложности процесса решения.

Все сказанное делает актуальной разработку более эффективного метода и соответствующего алгоритма решения линейных целочисленных оптимизационных задач: более надежного и обладающего меньшей вычислительной сложностью.

Базовой идеей комбинированного метода является композиция нескольких существующих методов. Линейное программирование [ REF _Ref5265035 \r \h 12, REF _Ref5264975 \r \h 28] можно использовать как целеуказание для методов перебора, тем самым достигается значительное сокращения области, в которой будет работать переборный метод – метод неявного перебора по векторной решетке. Именно работая в паре, эти два метода дадут выигрыш в производительности, по сравнению с самостоятельной работой.

Кроме того, в комбинированном методе запланирована возможность применения эвристик (если есть априорные данные о задаче), что позволить адаптировать комбинированной метод для конкретных задач, достигая еще большей производительности. Но при этом следует отметить, что предлагаемый подход по использованию эвристик не позволяет эвристическим алгоритмам влиять на универсальность метода, как это происходит при использовании эвристик в чистом виде. При проектировании метода сразу ставиться целью создание универсального метода, подходящего для широкого класса задач, а эвристики лишь служат для более быстрого нахождения оптимального решения [ REF _Ref5266695 \r \h 37].

Использование эвристических алгоритмов для улучшения свойств комбинированного метода

Иногда при решении оптимизационных задач имеет смысл использовать некоторую априорную информацию о решаемой задаче, если она есть, и если это позволить ускорить процедуру нахождения оптимального решения. В этом случае появляются так называемые эвристические алгоритмы, и существующие оптимизационные могут получать некоторые модификации, связанные с использованием эвристик.

Применение эвристических алгоритмов может приводить к тому, что оптимизационный метод теряет свою универсальность, что не является положительным моментом. При решении задачи по определению оптимальной топологии ЛВС будет использована эвристика, которая позволит значительно улучшить свойства комбинированного метода.

Одним из примеров использования эвристики в решении оптимизационных задач может являться специальный алгоритм определения допустимого решения транспортной задачи – метод минимального элемента. Несмотря на то, что транспортная задача относится к более широкому классу задач линейного программирования, чем целочисленные задачи, ее рассмотрение с точки зрения применения эвристик является хорошим примером.

В этом алгоритме используются некоторые идеи, например:

способ заполнения матрицы при процедуре пересчета, когда берется минимальное значение двух незаполненных перевозок;

тот факт, что порядок заполнения элементов в матрице жестко не задан,

которые не применимы для всех задач этого класса, а применяются лишь при решении транспортной задачи, так как используют конкретные свойства именно этой задачи. Но использование этого алгоритма в решении транспортной задачи гораздо выгоднее, чем применение общего подхода для поиска допустимого решения, так значительно быстрее и проще дает требуемый результат.

В комбинированном методе явным местом, где может применяться эвристика, является нахождение допустимого решения на втором этапе. Здесь эвристика может заменить применение метода неявного перебора по векторной решетке.

Свойства комбинированного метода

Комбинированный метод основан на комбинации возможностей линейного программирования, неявного перебора и применения эвристик, если это позволяет решаемая задача. Идея его состоит в последовательном проведении следующих четырех этапов вычислений. На первом этапе решается задача линейного программирования (ЗЛП), полученная из основной путем снятия условий целочисленности на переменные. Полученное оптимальное решение в случае его нецелочисленности становится отправной точкой для второго этапа реализации метода: поиска методом неявного перебора допустимого субоптимального решения исходной задачи в окрестности найденного на первом этапе нецелочисленного решения. На данном этапе можно применить эвристические алгоритмы более быстрого поиска допустимого решения, чем применение метода неявного перебора по векторной решетке, если это позволяют условия задачи. Третий этап состоит в формировании на основе целевой функции исходной задачи и найденного субоптимального решения так называемого фильтрующего ограничения, отсекающего от области допустимых решений ЗЛП большую часть с худшими решениями, чем найденное субоптимальное. Это ограничение вводится в структуру данных, сформированной и решенной на первом этапе ЗЛП. С использованием возможностей линейного программирования на этом же этапе определяются границы, отсеченной фильтрующим ограничением подобласти допустимых решений исходной задачи, которая содержит все лучшие решения, чем найденное субоптимальное (если, конечно, они существуют). Сформированная таким образом подобласть с установленными ее границами является основой для определения малой области для окончательного поиска оптимального решения исходной задачи методом неявного перебора на заключительном четвертом этапе. Таким образом, гарантируется нахождение оптимального решения исходной задачи, если оно существует. Но имеется возможность остановится и на найденном субоптимальном решении, оценив сверху возможную его погрешность по значению целевой функции. Использование метода неявного перебора на втором и четвертом этапах поиска не должно приводить к большой вычислительной сложности, так как гарантируется малый размер областей поиска в обоих случаях. Достоинством метода по сравнению с методами правильных отсечений и ветвей и границ является исключение влияния накопления ошибок при решении ЗЛП на надежность поиска. Априорный анализ показывает, что преимущества предлагаемого комбинированного метода будут возрастать при увеличении размерности решаемой задачи. Кроме этого, в комбинированном методе есть возможность применения различных эвристических алгоритмов для более точной локализации области перебора после первого этапа метода.

Язык программирования

Кроме концептуальных и алгоритмических особенностей, которые придали комбинированному методу его отличительные свойства, немаловажную роль сыграло качество его программной реализации. В задачах большой размерности очень сложно вручную проверить результат работы того или иного метода, поэтому на первый план выходит качество самого программирования. Программирование с использованием объектно-ориентированного анализа и проектирования позволяет избежать большинства потенциальных ошибок уже на стадии создания программы. На отладку и тестирование остается выявление не мелких ошибок, появившихся в процессе написания программы, а проверка глобальных идей, заложенных в алгоритме комбинированного метода. Программная метода была выполнена на языке C++ [ REF _Ref22928868 \r \h 9, REF _Ref22928890 \r \h 47, REF _Ref22928892 \r \h 49]. Созданная иерархия классов позволила разделить алгоритм на части, которые программировались и отлаживались независимо друг от друга. Являясь комбинированным методом, предлагаемый метод использует в себе несколько классических оптимизационных методов, например, симплекс-метод, метод неявного перебора по векторной решетке. При данном подходе, все используемые методы можно легко выносить в самостоятельные программы для тех или иных экспериментов. При необходимости расширения функциональности того или иного модуля сокращена к минимуму опасность нарушить работоспособность других модулей. Кроме того, в дальнейшем, при развитии метода или создания на его базе нового, будет возможность использовать уже созданные классы как базу для новой программной реализации. Синтаксис же языка C++ позволил написать программный код надежным и в тоже время ясным для дальнейшего сопровождения. Одной из проблем в программировании подобных задач (когда используются большие объемы памяти для хранения данных) является контроль использования динамической памяти. Механизмы автоматического создания и уничтожения объектов в языке С++ максимально облегчили эту задачу. Язык С++, являясь языком программирования, в полной мере реализующего парадигму объектно-ориентированного анализа и проектирования, позволил создать программную реализацию комбинированного метода, отвечающую стандартам современного программного обеспечения.

Сравнение альтернативных методов поиска оптимального решения математической модели линейного программирования

Критерии сравнения

Одной из основных проблем математического программирования является высокая вычислительная сложность методов оптимизации. Особенно это относится к целочисленному программированию, существующие методы которого имеют экспоненциально возрастающую вычислительную сложность при увеличении размерности задачи по числу оптимизационных переменных. В связи с этим в качестве основного показателя критерия сравнения выбирается машинное время нахождения оптимального решения. Для его фиксации в разработанные программы включены соответствующие средства.

Конечный результат оптимизации у каждого из сравниваемых методов потенциально один и тот же – оптимальное решение задачи (если, конечно, оно существует; возможным исходом является пустота области допустимых решений). Однако, как уже говорилось выше, некоторые из сравниваемых методов имеют недостатки, определяемые погрешностями машинных вычислений. Это прежде всего касается МПО и в значительно меньшей степени МВГ. Накопление погрешности может привести к пропуску оптимального решения, а в худшем случае – к аварийному сбою в работе алгоритма оптимизации. Поэтому в качестве следующего показателя сравнения выберем надежность поиска оптимального решения, которую можно оценить по частоте сбоя программы из-за накопленной погрешности и частоту пропуска оптимального решения. Эти частоты могут быть объективно определены на множестве решаемых задач, так как среди сравниваемых есть методы (МНПВР, КМ), принципиально свободные от указанных недостатков.

Все сравниваемые методы потенциально могут находить точное оптимальное решение. Но иногда из-за временных ограничений или других соображений у пользователя может возникнуть потребность поиска хотя бы какого либо допустимого решения (лучше субоптимального), но как можно быстрее. Поэтому свойство алгоритма оптимизации давать допустимое решение оптимизационной задачи до окончания полного ее решения да еще, если это возможно, с оценкой его точности является важным показателем метода [ REF _Ref22929670 \r \h 54].

Генератор линейных задач целочисленного программирования

Цели исследования требуют генерации большого числа разнообразных задач возрастающей размерности по числу оптимизационных переменных и ограничений, размерам гиперпараллепипеда полного перебора, имеющих различную конфигурацию области допустимых решений и направление по отношению к ней градиента целевой функции. Формирование множества таких задач практически невозможно без автоматизации. Для этого создан программный генератор [ REF _Ref22929716 \r \h 20], который работает следующим образом.

Ограничения определяются в виде неравенств типа “ EMBED Equation.3  ” с положительными правыми частями. Формирование ограничений осуществляется путем генерации псевдослучайных отношений правой части ограничения к коэффициентам при каждой переменной. Затем по сгенерированным отношениям в соответствии с определенными в программе значениями правых частей вычисляются и сами коэффициенты ограничений. Ограничения могут формироваться двух типов: наиболее распространенные, так называемые, ресурсные, у которых все коэффициенты в левой части являются неотрицательными числами; ограничения не ресурсного типа, в которых при отдельных оптимизационных переменных коэффициенты могут быть отрицательными числами. Тип ограничения формируется стохастически по заданной вероятности включения в формируемое ограничение отрицательных коэффициентов при переменных.

Коэффициенты целевой функции генерируются как независимые равномерно распределенные на отрезке  EMBED Equation.3   псевдослучайные величины.

Важной характеристикой сформированной задачи, влияющей на сравнительный анализ различных методов оптимизации (особенно, методов неявного перебора) является так называемая «жесткость» ограничений, определяющая величину той части гиперпараллепипеда полного перебора, которая занята областью допустимых решений. Для оценки сформированной задачи по этому показателю в генератор включен имитатор  EMBED Equation.3  -мерного дискретного равномерного распределения в гиперпараллепипеде полного перебора, с помощью которого по методу Монте-Карло оценивается указанный показатель.

Сравнение по надежности поиска оптимального решения.

Временные рамки проведения машинного эксперимента не позволили в полной мере провести сравнение по этому показателю. Априорный анализ алгоритмов показывает, что надежность МНПВР и КМ вне подозрений. Дело в том, что переборные методы, к которым относится и МНПВР, вообще не подвержены ошибкам, связанным с накоплением погрешностей вычислений, так как в них отсутствуют процедуры работы с действительными числами. КМ на основных этапах поиска решения (втором и четвертом) использует именно МНПВР, а возможные ошибки вычислений на первом и третьем этапах при работе алгоритма симплекс-таблиц не могут принципиально сказаться на результатах поиска: на этих этапах определяются лишь границы поиска для следующих за ними этапов КМ и ошибки, в крайнем случае, могут привести лишь к некоторому расширению области поиска и, соответственно, увеличению времени решения задачи.

В то же время, использование алгоритмов симплекс-таблиц в МПО и МВГ принципиально отличается от КМ тем, что там эти алгоритмы применяются непосредственно для нахождения оптимального решения. Накапливаемые при многократном применении алгоритмов симплекс-таблиц погрешности при выполнении процедур выявления целочисленных компонент решения, выделения дробных частей действительных чисел могут привести к принципиальным ошибкам: пропуску оптимального решения или вообще к аварийному сбою в работе алгоритма. В этом смысле МВГ обладает некоторыми преимуществами по сравнению с МПО из-за простоты вводимых новых ограничений при ветвлении. Для повышения надежности МПО, как уже говорилось выше, в него включены специальные процедуры оценки накопленной погрешности вычислений и использования ее для уменьшения вероятности возможных ошибок.

Приведенный анализ ограниченно подтвердился при проведении машинных экспериментов: в двух задачах из множества, на которых тестировался и исследовался МПО произошел пропуск оптимального решения с нахождением решения, более худшего по сравнению с оптимальным по значению целевой функции.

Сравнение по возможности досрочного результативного завершения работы алгоритма.

По этому показателю наибольшими преимуществами по сравнению с другими методами обладают МВГ и КМ, в которых для найденных на ранних этапах допустимых решений очень просто получаются оценки сверху возможного их отклонения от оптимального по значению целевой функции близкие к допустимому решению. Затем следует МНПВР, где допустимое решение может быть получено задолго до окончания работы алгоритма, но без оценки его отклонения от оптимального. Наихудшим по рассматриваемому показателю является МПО, в котором до окончательного завершения работы алгоритма не определяется ни одного допустимого решения исходной задачи.

Результаты решения задачи

При решении задачи были выполнены все четыре этапа КМ. Общее время решения задач – 6 часов на ЭВМ класса Pentium 200 MMX. На втором этапе было найдено субоптимальное решении со значением целевой функции 360. Это на 21.6% хуже оптимального равного 280. Объем перебора этом этапе составил 0.006% от полного. Дополнительный объем перебора на четвертом этапе составил 0.002%. Из этого видно, что концепция комбинированного метода эффективно реализовалась на этой задаче.

Кроме этого была произведена попытка решения этой задачи методом неявного перебора по векторной решетке, методом ветвей и границ, методом правильных отсечений. Но попытка не увенчалась успехом, так как перечисленные методы крайне долго решали задачу, и, как следствие этого, завершить решение задачи ими не удалось в процессе поставленного счетного эксперимента.

Заключение

Разработан, алгоритмически и программно реализован комбинированный метод решения задач целочисленного программирования, эффективно объединяющий возможности линейного программирования, неявного перебора по векторной решетке, а в определенных условиях и эвристические подходы для нахождения оптимального решения.

Осуществлены формализованная постановка задач определения оптимальной структуры ЛВС в виде линейных задач целочисленного программирования и их решение с использованием разработанного комбинированного метода.

Проведено сравнение комбинированного метода с рядом существующих универсальных методов решения задач рассматриваемого класса и получены следующие результаты:

комбинированный метод обладает преимуществами по сравнению с существующими методами, в частности, отсутствует влияние погрешности вычислений на надежность и точность оптимизации, в отличие от метода ветвей и границ, метода правильных отсечений отсутствует неопределенность, связанная с размерностью используемых структур данных и этапностью решения задачи;

существует возможность включения в работу метода эффективных эвристик, которые могут позволить существенно снизить вычислительную сложность поиска оптимального решения;

комбинированный метод превосходит существующие методы по затратам машинного времени на поиск оптимального решения, которые возрастают при увеличении размерности решаемой задачи.

Произведено обобщение существующего метода неявного перебора по векторной решетке, ориентированного на решение комбинаторных задач, для линейных задач целочисленного программирования общего вида. Это позволило эффективно использовать его в составе комбинированного метода, а также использовать его как самостоятельный метод поиска оптимального решения рассматриваемого класса задач.
