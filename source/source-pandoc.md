# Введение

Актуальность темы исследования

Современные системы поддержки принятия решения и автоматизированные
системы управление построены на последних достижениях прикладных наук.
Одной из таких наук является математическое программирование
\[1,30,32,33,44\]. С помощью математического программирования решаются
разнообразные задачи, так или иначе являющиеся составными частями
автоматизированных систем управления (АСУ), систем автоматизированного
проектирования (САПР) и систем поддержки принятия решения (СППР).

Важным подклассом задач математического программирования являются
линейные задачи целочисленного программирования
\[12,14,15,28,34,36,46,53,61,62\]. С помощью них формализуются многие
реальные задачи оптимального планирования \[6,7,8,13,19,24,29,31,35,52,
58,59\] и, в частности, такие типовые постановки задач этого класса,
как: основная производственная задача с дискретным характером
производимой продукции, задача о ранце, задача о загрузке транспорта и
многие другие.

Необходимо отметить, что многие задачи нелинейного программирования
могут быть сведены к линейным задачам.

В настоящее время существует ряд оптимизационных методов, позволяющих
решать задачи этого класса, такие как: методы неявного перебора, метод
правильных отсечений, метод ветвей и границ. Каждый из этих методов
имеет свои серьезные недостатки, ограничивающие их применение к решению
высокоразмерных оптимизационных задач, что делает актуальной разработку
более эффективного метода и соответствующего алгоритма решения линейных
целочисленных оптимизационных задач: более надежного и обладающего
меньшей вычислительной сложностью \[11,39,40,45,50,55\].

Цель работы

Целью работы является разработка комбинированного метода для решения
линейных задач целочисленного программирования: обоснование, разработка
алгоритмической и программной реализаций, исследование его свойств, и
решение с помощью него задачи по нахождению оптимальной топологии ЛВС.

Задачи исследования:

- решить с помощью разработанного комбинированного метода задачу по
  нахождению оптимальной топологии ЛВС.

- обосновать идеи и концепцию разработанного комбинированного метода
  решения линейных задач целочисленного программирования;

- разработать алгоритмическую реализацию комбинированного метода и
  эффективную программную реализацию комбинированного метода;

- исследовать свойства комбинированного метода и сравнить его с
  существующими методами, которые решают задачи этого же класса.

Методика исследования

В работе применяются методы системного анализа, в том числе
математического программирования, машинного эксперимента и
объектно-ориентированного проектирования программ.

Результаты, выносимые на защиту:

- результаты решения задачи по нахождению оптимальной топологии ЛВС при
  помощи комбинированного метода;

- комбинированный метод для решения задач линейного целочисленного
  программирования;

- результаты исследования свойств комбинированного метода и его
  результаты сравнения с существующими методами.

Научная новизна

- впервые предложена концепция комбинированного метода решения линейных
  задач целочисленного программирования, объединяющая методы линейного
  программирования, неявного перебора по векторной решетке и
  эвристические подходы для эффективного решения задач большой
  размерности. Использование эвристических подходов в комбинированном
  методе не влияет на возможность нахождения строго оптимального
  решения, а лишь обеспечивает более быстрый его поиск;

- проведено всестороннее сравнение разработанного метода с различными
  известными универсальными методами решения линейных задач
  целочисленного программирования (в том числе и с использованием
  машинного эксперимента), показавшее его преимущества;

- поставлены и решены задачи оптимизации топологии ЛВС с использованием
  разработанного метода.

Практическая значимость

В результате выполнения диссертационной работы автором получены
следующие результаты:

- с использованием объектно-ориентированного подхода осуществлена
  программная реализация комбинированного метода, позволяющая
  подстраивать алгоритм метода под конкретную задачу с помощью
  использования разнообразных эвристик, и с помощью этой программной
  реализации получено решение задачи определения оптимальной топологии
  локальной вычислительной сети;

- разработана эффективная библиотека современных оптимизационных методов
  для решения линейных задач целочисленного программирования;

- результаты, полученные в работе, внедрены на предприятии РКК
  «Энергия». С помощью разработанного программного обеспечения была
  решена задача по определению оптимальной топологии ЛВС этого
  предприятия. Кроме того, осуществлено внедрение разработанного
  программного обеспечения в учебный процесс МАИ в составе лабораторного
  комплекса по изучению методов решения задач линейного целочисленного
  программирования.

Разработанное программное обеспечение зарегистрировано в Реестре
программ для ЭВМ \[21\].

Апробация работы

Идеи и концепции, предложенные в диссертационной работе, докладывались и
обсуждались на различных конференциях, в том числе международных, в
частности, на 5, 6, 7, 8 и 11 международных научно-технических семинарах
«Современные технологии в задачах управления автоматики и обработки
информации» (Россия, Алушта, 1996, 1997, 1998, 1999, 2002), на восьмой
международной научно-технической конференции студентов и аспирантов
«Радиоэлектроника, электротехника и энергетика» (Москва, МЭИ, 2002).

Разработка комбинированного метода осуществлена в рамках плановой НИР ПН
235 (302-98-05) кафедры 302 МАИ, выполненной на основе конкурса грантов
Министерства образования в 1998-1999 г.

Результаты работы отражены в 11 публикациях.

#### Формализованная запись задачи

Предположения

- Если в оконечном узле дерева (по плану прокладки соединительных
  кабелей) находится более одного пользователя, то в этом узле ставится
  концентратор, а не прокладываются параллельные кабели к каждому
  пользователю от предыдущего концентратора.

- Задан асимметричный орграф:\
  ![](media/image5.wmf)\
  (между непосредственно связанной парой) из вершин $(O'O^{''})$
  существующей дуги $d' = \lbrack v',v^{''}\rbrack$ и
  $d^{''} = \lbrack v',v^{''}\rbrack$.

- По информации о количестве комнат (вершин) составляется множество всех
  возможных дуг $D$. При этом множество $D$ разбивается на два
  подмножества:

  - $D_{V}^{+}$ -- подмножество дуг, входящих в вершину;

  - $D_{V}^{-}$ -- подмножество дуг, выходящих из вершины.

Параметры задачи

- $V$ -- множество комнат здания ($|V| = m$);

- $D$ -- множество всех возможных дуг;

- $S$ -- номер комнаты, где помещен ЦК;

- $\text{KP}\lbrack v\rbrack\text{   }(v = \overline{1,m})$ --
  количество пользователей, находящихся в каждой вершине $v$. При этом
  искусственно предполагается, что
  $\text{KP}\lbrack S\rbrack = - \text{KPS}$, где обще число
  пользователей ЛВС
  $\text{KPS} = \sum_{v \in V}^{}{\text{KP}\lbrack v\rbrack}$.

- $c_{k}$ -- стоимость отдельного концентратора;

- $a$ -- число портов каждого отдельного концентратора для подключения
  кабелей ЛВС;

- $c\lbrack d\rbrack$ - стоимость прокладки кабеля по дуге $d$. Для
  каждой дуги $d\lbrack\text{ij}\rbrack \in D$ определена ее длина
  $l_{d}$ (причем $l_{\text{ij}} = l_{\text{ji}}$), а с учетом стоимости
  прокладки единицы длины соединительных кабелей $c_{l}$, определена
  стоимость прокладки кабеля по дуге $d$:
  $c\lbrack d\rbrack = c\lbrack\text{ij}\rbrack = c\lbrack\text{ji}\rbrack$.
  Причем, если между комнатами $i$ и $j$ невозможно проложить кабель, то
  $c\lbrack d\rbrack = c\lbrack\text{ij}\rbrack = c\lbrack\text{ji}\rbrack = \infty$.

Оптимизационные переменные

- $x\lbrack d\rbrack$, $(d \in D)$ -- количество пользователей,
  использующих дугу $d$ для связи с ЦК;

- $z\lbrack d\rbrack$, $(d \in D)$ -- бинарная переменная
  $z\lbrack d\rbrack = 1$, если по дуге $d$ проложен кабель,
  $z\lbrack d\rbrack = 0$-- в противном случае;

- $y\lbrack v\rbrack$, $(v \in V)$ -- количество концентраторов,
  объединенных в стек, расположенных в вершине $V$ орграфа. Общее число
  портов в вершине $V$ орграфа, которое может быть использовано для
  организации связи с другими вершинами равно
  $a \cdot y\lbrack v\rbrack$.

Критерии оптимизации

Минимум затрат на прокладку кабелей и установку концентраторов:

$\underset{x,z,y}{\text{min}}\sum_{d \in D}^{}{c\lbrack d\rbrack \cdot z\lbrack d\rbrack +}\sum_{v \in V}^{}{c_{k}y\lbrack v\rbrack}$.

Ограничения задачи

1.  Балансовые ограничения по загрузке дуг ЛВС:\
    $\sum_{d \in D_{V}^{+}}^{}{x\lbrack d\rbrack} - \sum_{d \in D_{V}^{-}}^{}{x\lbrack d\rbrack} = \text{KP}\lbrack v\rbrack\text{  }(v \in V)$

2.  Ограничение, определяющее согласованное изменение переменных
    $x\lbrack d\rbrack$ и $z\lbrack d\rbrack$:
    $x\lbrack d\rbrack \leq \text{KPS} \cdot z\lbrack d\rbrack\text{  }(d \in D)$

3.  Ограничение на необходимость установки концентраторов:\

## Методы решения задач целочисленного программирования и подходы к решению поставленной задачи

Целочисленные линейные задачи являются важным подклассом задач
дискретного программирования. К этому подклассу относятся такие типовые
содержательные постановки оптимизационных задач, как: основная
производственная задача с дискретным характером производимой продукции,
задача о ранце, задача о загрузке транспорта, различного вида задачи о
назначениях и многие другие.

Несвязность области допустимых решений этого класса задач порождает
дополнительные по сравнению с задачами линейного и нелинейного
программирования \[5,41,56,57,60\] проблемы разработки методов их
решения: невозможность в полной мере использования итеративных процедур
непрерывного движения в пространстве оптимизационных переменных;
неправомерность в общем случае применения подхода, основанного на
округлении наилучшего нецелочисленного решения и др. Указанные
особенности линейных задач целочисленного программирования приводят к
тому, что алгоритмическая и вычислительная сложность методов их решения
является высокой.

Кроме того, существует более широкий класс задач, таких как задачи
частично-целочисленного программирования. При их решении могут
использоваться методы решения линейных целочисленных задач как составная
часть процесса поиска оптимального решения. Одним из методов решения
задач частично-целочисленного программирования является метод
декомпозиции. В этом методе происходит разделение задачи на две
подзадачи: задачу линейного программирования и целочисленную задачу,
которую после этого приходится многократно решать, поэтому и в
частично-целочисленном программировании наличие эффективных методов
решения целочисленных задач имеет большое значение.

### Существующие методы решения линейных задач целочисленного программирования

#### Методы неявного перебора

Методы этого класса применяются для решения задач целочисленного
программирования. Множество решений большинства задач целочисленного
программирования являются не только счетным, но и конечным. Но
вычислительная сложность полного перебора этого множества очевидно
высока. Разработаны более эффективные методы перебора, позволяющие
уменьшить объем вычислений либо за счет более эффективной организации
работы с каждой отдельной точкой множества решений, либо за счет
сокращения числа рассматриваемых точек. Эти методы могут использовать
априорные сведения о решаемой задаче, а также информацию, получаемую
непосредственно в процессе перебора. Их называют методами неявного
перебора. Наиболее существенно различные группы этих методов отличаются
способом организации собственно процедуры перебора, что и определяет их
названия.

##### Метод лексикографического перебора

Этот метод основан на процедуре перебора отдельных точек области
возможных решений, аналогичной упорядочению слов в словаре и называемой,
поэтому, лексикографической. Все переменные исходной задачи. Все
переменные исходной задачи упорядочиваются по быстроте изменения. Если
никаких дополнительных соображений при этом не использовать, то этот
порядок будет соответствовать естественному порядку нумерации переменных
исходной формализованной записи оптимизационной задачи: самой
медленноменяющейся переменной будет $x\lbrack 1\rbrack$, а самой
быстроменяющейся - $x\lbrack n\rbrack$. В процессе перебора каждая
переменная $x\lbrack j\rbrack\text{   }(j = \overline{1,n})$ принимает
последовательно целочисленные значения с $x_{j}^{\text{min}}$ (обычно
$x_{j}^{\text{min}} = 0$) и кончая $x_{j}^{\text{max}}$. Но прежде, чем
увеличивать значение переменной $x\lbrack j\rbrack$ на единицу,
необходимо перебрать все возможные комбинации переменных
$x\lbrack j + 1\rbrack,\ldots,x\lbrack n\rbrack$. Для этого каждая из
них должна пробежать все свои значения от наименьшего до наибольшего
возможного.

Недостатком этого метода можно считать тот факт, что область перебора
при больших размерностях задач будет максимальна, что приводит к
колоссальной вычислительной сложности.

##### Метода неявного перебора по векторной решетке

Векторной решеткой называют математическую структуру, определяющую
расположение и взаимосвязь различных значений (точек) целочисленного
$n$-мерного вектора $x = (x\lbrack 1\rbrack,\ldots,x\lbrack n\rbrack)$,
у которого каждая компонента $x\lbrack j\rbrack$ может принимать
целочисленные значения, начиная с $x_{j}^{\text{min}}$ и кончая
$x_{j}^{\text{max}}$. Отдельные точки вектора $x$ располагаются по
уровням Хэмминга, определяемые использованием понятия расстояния
Хэмминга.

Расстоянием Хэмминга называют местоположение точек векторной решетки,
расстояние Хэмминга для которых от точки
$(x_{1}^{\text{min}},\ldots,x_{n}^{\text{min}})$ - одно и то же. При
этом номер уровня равен величине расстояния Хэмминга. Как правило,
предполагается, что все
$x_{j}^{\text{min}} = 0\text{  }(j = \overline{1,n})$.

Формирование векторной решетки завершает правило, согласно которому
каждые две ее точки, находящиеся на соседних уровнях и отстоящие друг
относительно друга на расстоянии Хэмминга, равном единице, соединяются
дугой.

Перебор точек вектора $x$ с использованием векторной решетки
осуществляется по ее маршрутам с помощью шагов вперед и назад.

Так как метод неявного перебора по векторной решетке перебирает
недопустимые точки, то объем перебора напрямую связан с объемом области
недопустимых решений по отношению к полному перебору, и некоторых
задачах, где допустимых точек мало, может получиться, что методу
придется перебирать значительный объем точек.

В данное работе предлагается обобщение метода неявного перебора по
векторной решетке для задач, где оптимизационные переменные
$x\lbrack j\rbrack$ могут принимать значения $1 \div h\lbrack j\rbrack$.
В этом случае, комбинаторная постановка является частным случаем этого
обобщения, когда все $h\lbrack j\rbrack = 1$.

##### Метод неявного перебора, основанный на стратегии локального поиска

Идея метода заключается в оптимизации с использованием проб и ошибок.
Сущность его состоит в последовательном переходе от одной точки области
допустимых решений к другой путем просмотра множества решений, входящих
в окрестность первой точки, и выбора решения с лучшим значением целевой
функции. Процесс перехода (большая итерация метода) заканчивается, если
при поиске в окрестности очередной точки не найдено лучшего решения.
Таких больших итераций делается в общем случае несколько, начиная с
различных начальных допустимых точек. Принципиальные отличия такого
метода от рассмотренных выше методов заключаются в следующем:

- в данном методе в процессе перебора рассматриваются только допустимые
  точки;

- отсутствует гарантия нахождения точного оптимального решения исходной
  задачи (отсюда и название -- локальный поиск).

Явным недостатком этого метода является то, что процесс решения не
гарантирует, что оптимальное решение будет найдено, даже если оно
существует. Кроме того, метод не носит универсальный характер и должен
подстраиваться под каждую задачу, а комбинированный метод --
универсальный и гарантирует нахождение оптимального решения в
независимости от того, используются ли в нем эвристические алгоритмы,
или нет.

#### Метод отсекающих плоскостей

Этот метод применяется к задачам дискретного целочисленного и
частично-целочисленного программирования. Такие задачи при снятии
условия целочисленности на переменные превращаются в обычные задачи
линейного программирования (ЗЛП). Область допустимых решений ЗЛП
полученной таким образом, будет обозначать $D'$. Очевидно, что область
допустимых решений исходной дискретной задачи $D \subset D'$.

Сущность метода отсекающих плоскостей заключатся в следующем.

Вначале решается соответствующая задача линейного программирования на
область $D'$. По этим результатам делаются следующие выводы о решении
исходной задачи:

- если $D' = \varnothing$, то на основе того, что $D \subset D'$,
  делается вывод, что $D = \varnothing$;

- если целевая функция не ограничена на $D'$, то аналогичный вывод
  делается по исходной задаче. Это обосновывается тем, что в области,
  содержащей бесконечно удаленную точку, практически всегда можно найти
  аналогичного рода точку, принадлежащую области допустимых решений
  исходной задачи. Исключением может быть случай вырождения области $D'$
  в образующую, не проходящую ни через одну точку $D$, который в
  практических задачах вряд ли может реализовываться;

- если оптимальное решение задачи линейного программирования на $D'$
  принадлежит области допустимых решений $D$ исходной задачи, то оно
  является одновременно и оптимальным решением исходной задачи. Это
  также следует из того, что $D \subset D'$. На этом основании
  максимальное значение целевой функции задачи линейного
  программирования на $D'$ является всегда верхней границей для значения
  целевой функции исходной дискретной задачи, и тогда из их равенства
  следует сформулированный выше вывод;

- если оптимальное решение задачи линейного программирования на $D'$ не
  принадлежит области допустимых решений $D$, то осуществляется переход
  к второму этапу решения исходной задачи, реализующему собственно
  основную идею метода отсекающих плоскостей.

Этот этап состоит в последовательном решении получающихся друг из друга
ряда задач линейного программирования. Каждая последующая формируется из
предыдущей путем введения дополнительно к имеющимся ограничениям нового,
называемого правильным отсечением. Правильным оно называется потому, что
формируется таким образом, что отсекает от допустимой области предыдущей
задачи ее оптимальное решение, не отсекая в то же время ни одного
допустимого решения исходной задачи. Этот процесс прекращается, когда в
очередной задаче либо будет получено оптимальной решение, принадлежащее
$D$, либо будет установлена пустота области ее допустимых решений. В
этих случаях на основе свойств правильности введенных ограничений
сохраняют справедливость все рассуждения, приведенные в п. 2 и 4:
оптимальное решение расширенной задачи линейного программирования,
принадлежащее $D$, является оптимальным решением исходной задачи. Из
пустоты допустимой области расширенной задачи линейного программирования
следует пустота допустимой области исходной задачи.

Различные варианты методов правильных отсечений в основном отличаются
способами формирования этих отсечений.

Одним из таких алгоритмов является алгоритм Гомори.

Положительным свойством алгоритмов, реализующих методы правильных
отсечений, является то, что алгоритмы обладают свойствами определенности
и сходимости за конечное число итераций. Однако есть несколько
отрицательных свойств:

- Алгоритмам присуща большая вычислительная сложность: даже при
  небольших по размерности задачах число вводимых правильных отсечений и
  соответственно больших итерации может быть большим. Вычислительные
  эксперименты показали, что применение метода правильных отсечений для
  некоторых задач (например, комбинаторных) равносильно по
  вычислительной сложности полному перебору.

- Первое допустимое решение исходной задачи, если
  $D \neq \notin \varnothing$, находится в последний момент его работы
  (оно же является ее оптимальным решением).

- Свойство симплекс-таблиц, заключающееся в накоплении ошибок при
  итеративном выполнении преобразований Гаусса-Жордана с действительными
  числами, может проявиться здесь в большей степени. В дополнение к
  необходимости различать в обычном симплекс-методе отрицательные,
  положительные и нулевые элементы действительной части симплекс-таблицы
  здесь необходимо различать целые и дробные элементы, определять, по
  возможности, точнее дробную часть числа. Решение этой проблемы
  возможно путем перехода в методе симплекс-таблиц к работе с
  рациональными числами, однако при этом может возникнуть проблема
  нехватки разрядности компьютера для хранения целых чисел, являющихся
  числителями и знаменателями рациональных чисел.

#### Метод ветвей и границ

Этот метод является наиболее общим среди всех методов дискретного
программирования и не имеет принципиальных ограничений по применению.
Его сущности заключается в направлении ветвления \[27\] области
допустимых решений оптимизационной задачи с целью локализации
оптимального решения в одной области из ее подобластей. При применении
метода ветвей и границ к каждой конкретной задаче в первую очередь
должны быть определены две важнейшие его процедуры:

1.  ветвление множества возможных решений;

2.  вычисление нижних и верхних оценок целевой функции

Основным недостатком метода ветвей и границ, затрудняющим его
программную реализацию, является отсутствие данных о максимальном
количестве элементов множества номеров подмножеств $D_{i}$ (или
$D_{i}'$), находящихся на концах ветвей и ветвление которых еще не
прекращено, что в ряде случаев может приводить к переполнению памяти
компьютера.

#### Использование эвристических алгоритмов для улучшения свойств оптимизационных методов решения задач линейного программирования

Иногда при решении оптимизационных задач имеет смысл использовать
некоторую априорную информацию о решаемой задаче, если она есть, и если
это позволить ускорить процедуру нахождения оптимального решения. В этом
случае появляются так называемые эвристические алгоритмы, и существующие
оптимизационные могут получать некоторые модификации, связанные с
использованием эвристик.

Применение эвристических алгоритмов может приводить к тому, что
оптимизационный метод теряет свою универсальность, что не является
положительным моментом. При решении задачи по определению оптимальной
топологии ЛВС будет использована эвристика, которая позволит значительно
улучшить свойства комбинированного метода. Для понимания сути применения
эвристических алгоритмов в решении оптимизационных задач, необходимо
рассмотреть некоторые примеры.

##### Метод минимального элемента для поиска допустимого решения транспортной задачи

Одним из примеров использования эвристики в решении оптимизационных
задач может являться специальный алгоритм определения допустимого
решения транспортной задачи. Несмотря на то, что транспортная задача
относится к более широкому классу задач линейного программирования, чем
целочисленные задачи, ее рассмотрение с точки зрения применения эвристик
является хорошим примером.

В этом алгоритме используются некоторые идеи, например:

- способ заполнения матрицы при процедуре пересчета, когда берется
  минимальное значение двух незаполненных перевозок;

- тот факт, что порядок заполнения элементов в матрице жестко не задан,

которые не применимы для всех задач этого класса, а применяются лишь при
решении транспортной задачи, так как используют конкретные свойства
именно этой задачи. Но использование этого алгоритма в решении
транспортной задачи гораздо выгоднее, чем применение общего подхода для
поиска допустимого решения, так значительно быстрее и проще дает
требуемый результат.

### Предпосылки создания нового метода для решения линейных целочисленных задач

Таким образом, из предыдущего обсуждения выявлено, что каждый из
рассмотренных методов решения задач данного класса имеет свои
недостатки, ограничивающие их применение к решению высокоразмерных
оптимизационных задач \[55\]. В методах неявного перебора вычислительная
сложность растет экспоненциально с ростом размерности оптимизационной
задачи по числу переменных. Метод правильных отсечений (алгоритм Гомори)
имеет целый спектр недостатков. Это и отсутствие хотя бы одного
найденного допустимого решения до конца работы алгоритма, и накопление
ошибок вычислений при итерациях симплекс-метода и полная неясность с
числом больших итераций (количеством вводимых в задачу правильных
отсечений, причем, как правило, очень большим даже в задачах
сравнительно небольшой размерности). Последние два недостатка делают
работу алгоритма ненадежной. Ненадежность и вычислительная сложность
алгоритма резко возрастают с увеличением размерности задачи как по числу
переменных, так и по числу ограничений. Метод ветвей и границ
применительно к указанному классу задач порождает проблему хранения
данных по итогам решения задач линейного программирования на концах
ветвей: чтобы исключить повторное решение с самого начала задач
линейного программирования, получающихся в результате ветвления, и
применить двойственный вариант метода, необходимо для каждой задачи
хранить полную структуру исходных данных и заключительную
симплекс-таблицу. Отсутствие каких-либо оценок по количеству ветвлений
до полного решения задачи и порождает указанную проблему. Кроме того,
последовательное использование симплекс-метода после каждого ветвления
при большом числе ветвлений приводит, как и в случае алгоритма Гомори, к
накоплению ошибок вычислений с теми же самыми последствиями.
Альтернативой такому подходу в методах правильных отсечений и ветвей и
границ является решение задач линейного программирования на отдельных
итерациях методов с самого начала, но это с очевидностью приводит к еще
более резкому возрастанию вычислительной сложности процесса решения.

Все сказанное делает актуальной разработку более эффективного метода и
соответствующего алгоритма решения линейных целочисленных
оптимизационных задач: более надежного и обладающего меньшей
вычислительной сложностью.

Базовой идеей комбинированного метода является композиция нескольких
существующих методов. Линейное программирование \[12,28\] можно
использовать как целеуказание для методов перебора, тем самым
достигается значительное сокращения области, в которой будет работать
переборный метод -- метод неявного перебора по векторной решетке. Именно
работая в паре, эти два метода дадут выигрыш в производительности, по
сравнению с самостоятельной работой.

Кроме того, в комбинированном методе запланирована возможность
применения эвристик (если есть априорные данные о задаче), что позволить
адаптировать комбинированной метод для конкретных задач, достигая еще
большей производительности. Но при этом следует отметить, что
предлагаемый подход по использованию эвристик не позволяет эвристическим
алгоритмам влиять на универсальность метода, как это происходит при
использовании эвристик в чистом виде. При проектировании метода сразу
ставиться целью создание универсального метода, подходящего для широкого
класса задач, а эвристики лишь служат для более быстрого нахождения
оптимального решения \[37\].

# Разработка комбинированного метода для решения задачи определения оптимальной топологии ЛВС

## Необходимость комбинированного метода

Все существующие на данный момент метода решения задач линейного
целочисленного программирования имеют очень серьезные недостатки,
доходящие даже до того, то в процессе решения не гарантируется
нахождение решения или установления факта его отсутствия, а возможно
даже аварийное завершение процесса вычислений из-за накопленной
погрешности.

## Теоретическое обоснование комбинированного метода решения математической модели линейного целочисленного программирования

### Формализованная запись исходной задачи

Предполагается, что исходная оптимизационная задача имеет следующий вид:

$\underset{x}{\text{min}}Z = \text{cx   }D = \left\{ x \in N^{n}:\text{Ax}\underset{}{\overset{}{=}}b(2\text{.}1\text{.}1);0 \leq x \leq h(2\text{.}1\text{.}2) \right\}$,
(2.1)

где:

- $x$ - $n$-мерный вектор основных оптимизационных переменных;

- $c$ - $n$-мерная вектор-строка коэффициентов целевой функции, которые
  здесь, в исходной постановке могут быть любыми действительными
  числами; направление оптимизации на минимум не является в исходной
  постановке обязательным и определяется здесь таким образом, чтобы
  избежать введения дополнительных обозначений, так как одна из
  центральных составляющих метода оптимизации (алгоритм неявного
  перебора), как будет видно далее, требует преобразования задачи к
  такому виду;

- $N^{n}$- $n$-мерное множество целых неотрицательных чисел;

- \- основные ограничения задачи, записанные в матричной форме: $A$ -
  матрица $m \times n$; $b$ - $m$-мерный вектор-столбец;

- (2.1.2) - двусторонние ограничения на оптимизационные переменные;

- $h$ - $n$-мерный вектор-столбец ограничений на оптимизационные
  переменные сверху; (предполагается, что
  $h'\lbrack j\rbrack < \infty\text{  }(j = \overline{1,n})$, то есть
  каждая оптимизационная переменная имеет целочисленное ограничение
  сверху, хотя бы как оценку сверху на возможное максимальное ее
  значение; в дискретных задачах определение такой оценки не
  предоставляет сложности, во многих же задачах значения
  $h\lbrack j\rbrack$ диктуются исходной содержательной постановкой).

### Метод неявного перебора по векторной решетке и возможности его применения в комбинированном методе

#### Идеи неявного перебора по векторной решетке

В \[53\] изложены основные правила метода неявного перебора по векторной
решетке для частного случая целочисленных линейных задач комбинаторного
типа. В \[23\] на основе этих правил с некоторыми дополнениями,
исключающими при переборе повторный заход в одни и те же точки векторной
решетки, построен алгоритм оптимизации. Здесь проведено обобщение всех
правил алгоритма для общего случая целочисленной линейной задачи вида
(2.1).

Перебор точек вектора $x$ с использованием векторной решетки
осуществляется по ее маршрутам с помощью шагов вперед и назад.

Маршрутом называется любой путь, проходящий через вершины векторной
решетки, соединенными дугами. Шаг вперед - это переход на уровень с
большим номером, а шаг назад - обратное движение. Так как каждый шаг
вперед связан с увеличением соответствующей переменной на единицу, то
это перемещение определяется как «шаг вперед по $j$-й переменной». Чтобы
при движении вперед исключить повторное рассмотрение точек, для каждой
точки $x$ множество переменных $J_{x}$, по которым из этой точки можно
сделать шаги вперед:

$J_{x} = \left\{ j \in \overline{1,n}:j \geq j_{x} \right\}$, (2.2)

где
$j_{x} = \underset{\left\{ j \in \overline{1,n}:x\lbrack j\rbrack \neq 0 \right\}}{\text{max}}j$,
(2.3)

причем переменная $j = j_{x}$ включается в $J_{x}$ только в том случае,
если$x\lbrack j_{x}\rbrack < h\lbrack j_{x}\rbrack$.

Точки, для которых $x\lbrack n\rbrack = h\lbrack n\rbrack$ будем
называть конечными. После перемещения в такую точку с помощью шага
вперед обязательно должен следовать шаг назад.

Движение по векторной решетке начинается и заканчивается в нулевой
точке. Завершение перебора в нулевой точке происходит тогда, когда все
шаги вперед из нее уже рассмотрены.

Ниже рассматриваются правила, применение которых позволяет сократить
число перебираемых решений (точек) и, следовательно, сделать перебор
неявным. Применение первого правила связано с необходимостью
эквивалентного преобразования исходной задачи к виду, для которого
направление оптимизации - минимизация, а все коэффициенты целевой
функции являются неотрицательными. Если в исходной задаче, приведенной к
поиску минимума целевой функции, некоторый ее коэффициент при $j$-й
переменной $c\lbrack j\rbrack < 0$, то необходимо сделать следующую
замену переменных:

$x\lbrack j\rbrack = h\lbrack j\rbrack - x'\lbrack j\rbrack$, (2.4)

При такой замене, во-первых,
$c'\lbrack j\rbrack = - c\lbrack j\rbrack > 0$, (то есть достигается
основная цель замены); во-вторых, образующаяся при такой замене
константа может быть удалена из целевой функции, так как она не
оказывает влияния на результат оптимизации. Очевидно, что эту замену для
обеспечения эквивалентности преобразования необходимо сделать не только
в целевой функции, но и во всех ограничениях.

В задаче, у которой все $c\lbrack j\rbrack > 0$, при движении по
векторной решетке с помощью шагов вперед целевая функция возрастает, то
есть ее значение с точки зрения оптимизации ухудшается. На этой основе
формулируется следующее правило, ограничивающее движение по векторной
решетке: как только при перемещении по векторной решетке достигается
допустимая точка, дальнейшее движение с помощью шагов вперед
прекращается и делается шаг назад.

Из этого правила следует, что алгоритм работает в основном с
недопустимыми решениями. В частности, если допустимой оказывается
нулевая точка, то она является наилучшим допустимым и, следовательно,
оптимальным решением всей задачи.

Кроме того, для сокращения перебора по векторной решетке формулируется
еще ряд правил, которые названы критериями: критерий недопустимости
(КH), критерий планомерного исключения альтернатив (КПИА), критерий
предпочтительной переменной (КПП).

#### Критерий недопустимости

Этот критерий применяется к каждой точке векторной решетки после
первоначального попадания в нее при движении с помощью шагов вперед. Он
определяет условия невозможности достижения при движении вперед из
данной точки ни одного допустимого решения. Если эти условия
выполняются, то говорят, что критерий в данной точке сработал. В этом
случае необходимо закончить движение по векторной решетке с помощью
шагов вперед и сделать шаг назад. Ниже определяются дополнительные
преобразования исходной задачи и новые понятия и обозначения, которые
необходимы для формализованного описания действия КН.

Исходная задача приводится к форме, где все ограничения имеют вид
неравенства вида "($$)":

$\sum_{j = 1}^{n}{a\lbrack\text{ij}\rbrack x\lbrack j\rbrack \leq b\lbrack i\rbrack,\text{  }(i = \overline{1,m})}$,
(2.5)

Если в исходной задаче есть ограничения типа равенства, то эквивалентным
преобразованием является замена этого равенства на два соответствующих
неравенства противоположного типа: одно - типа "$$", а другое - типа
"$$".

В каждое $i$-е ограничения вводится балансовая переменная
$y\lbrack i\rbrack\text{  }(i = \overline{1,m})$, на которую
накладывается условие неотрицательности ($y\lbrack i\rbrack \geq 0$).
Получается следующая эквивалентная система ограничений:

$\sum_{j = 1}^{n}{a\lbrack\text{ij}\rbrack x\lbrack j\rbrack + y\lbrack i\rbrack = b\lbrack i\rbrack,\text{  }y\lbrack i\rbrack \geq 0,\text{  }(i = \overline{1,m})}$,
(2.6)

Обозначим через $y_{x}\lbrack i\rbrack$ значение балансовой переменной
$y\lbrack i\rbrack$, вычисленное для фиксированной точки $x$. Тогда,
если в некоторой точке $x$ $y_{x}\lbrack i\rbrack < 0$, то $i$-e
ограничение исходной задачи не выполняется. Если же все
$y_{x}\lbrack i\rbrack \geq 0\text{  }(i = \overline{1,m})$, то
указанная точка $x$является допустимым решением задачи.

Вводятся в рассмотрение два множества:

$I_{x}$ - множество невыполняющихся в точке $x$ ограничений;

$J^{x_{i}}$ - множество номеров переменных, по которым из точки $x$ еще
можно сделать шаг вперед, и при которых коэффициенты в $i$-м ограничении
отрицательны.

Формальное определение этих множеств:

$I_{x} = \left\{ i \in \overline{1,m}:y_{x}\lbrack i\rbrack < 0 \right\}$,
(2.7)

$J^{x_{i}} = \left\{ j \in J_{x};a\lbrack\text{ij}\rbrack < 0 \right\}$,
(2.8)

Очевидно, что осуществление шага вперед по переменной
$j \in J^{x_{i}}(i \in I_{x})$ увеличивает балансовую переменную
$y\lbrack i\rbrack$. С учетом этих соображений и введенных понятий
формальное определение действия КН осуществляется следующим образом:
если хотя бы для одного из невыполняющихся в рассматриваемой точке $x$
ограничений $i \in I_{x}$ выполняется условие:

$\sum_{j \in J^{x_{i}}}^{}{a\lbrack\text{ij}\rbrack(h\lbrack j\rbrack - x\lbrack j\rbrack) > y_{x}\lbrack i\rbrack}$,
(2.9)

то точка $x$отсеивается по КH.

Обоснование этого правила состоит в том, что, даже максимально увеличив
значения всех переменных $j \in J^{x_{i}}$, невозможно обеспечить такое
увеличение $y\lbrack i\rbrack$, при котором она станет неотрицательной,
то есть. невозможно при движении вперед добиться выполнения $i$-го
ограничения.

#### Критерий планомерного исключения альтернатив

Этот критерий применяется к отдельным шагам вперед из рассматриваемой
точки после того, как найдено хотя бы одно допустимое решение исходной
задачи. Сущность его работы состоит в том, что он отсеивает те шаги
вперед, которые не могут привести к лучшим решениям (меньшим по значению
целевой функции). Действие КПИА основано на использовании фильтрующего
ограничения:

$\sum_{j = 1}^{n}{c\lbrack j\rbrack x\lbrack j\rbrack < z(x_{\partial})}$,
(2.10)

где $x_{\partial}$- лучшее найденное к данному моменту допустимое
решение исходной задачи.

После введения балансовой переменной $y\lbrack 0\rbrack$ фильтрующие
ограничения преобразуется к виду:

$\sum_{j = 1}^{n}{c\lbrack j\rbrack x\lbrack j\rbrack + y\lbrack 0\rbrack = z(x_{\partial})}$,
(2.11)

Формальное правило КПИА, применяемое к шагу вперед по $j$-й переменой из
точки $x$, состоит в проверке неравенства:

$c\lbrack j\rbrack \geq y_{x}\lbrack 0\rbrack$, (2.12)

Если это неравенство выполняется, то шаг вперед по $j$-й переменной из
точки $x$ отсеивается, так как движения по данному маршруту не может
привести к меньшему значению целевой функции, чем ранее найденное для
лучшего допустимого решения.

Обоснование этого правила основано на следующих рассуждениях. Пока
рассматриваются решения, для которых выполняется (2.10), то
$y\lbrack 0\rbrack > 0$. При осуществлении каждого шага вперед по $j$-й
переменной значение $y\lbrack 0\rbrack$ уменьшается на величину
$c\lbrack j\rbrack$ и при выполнении (2.12) может стать неположительным,
откуда и следует вывод КПИА.

#### Критерий предпочтительной переменной

Назначение этого критерия --- упорядочение шагов из каждой недопустимой
точки, в которой не сработал КH, после первого прихода в нее. Цель
упорядочения - рассматривать в первую очередь те маршруты, которые
приводят к допустимым решениям. Действие КПП по сокращению перебора,
если цель упорядочения достигается, реализуется через совместную работу
с КПИА: чем раньше будет найдено первое допустимое решение, тем раньше
включится в работу КПИА; при нахождении затем лучших допустимых решений
действие фильтрующего ограничения по отсеву шагов вперед становится
более жестким. Формальное правило КПП, применяемое к возможным шагам
вперед их точки $x$ ($j \in J^{x_{i}}$), состоит в их упорядочении по
возрастанию значений следующих сумм:

$\sum_{i \in I_{x}}^{}{a\lbrack\text{ij}\rbrack(h\lbrack j\rbrack - x\lbrack j\rbrack)}$,
(2.13)

где $I_{x}$ формируется на основании (2.7).

Идея применения критерия состоит в том, чтобы в первую очередь
рассматривать маршруты, не приводящие к сильному убыванию балансовых
переменных $y\lbrack i\rbrack$ в невыполняющихся ограничениях, а
приводящие скорее к их возрастанию (при отрицательных
$a\lbrack\text{ij}\rbrack$), то есть к сокращению множества $I_{x}$.
Действие КПП в отличие от КH и КПИА носит эвристический характер и не
всегда может привести к желаемым результатам, так как выбор маршрута по
наименьшему значению указанных выше сумм не гарантирует первоочередное
движение к допустимым решениям.

Укрупненная структура алгоритма метода осталась такой же, как и для
задачи с бинарными переменными. Этот алгоритм приведен в \[53\].

#### Новые возможности неявного перебора по векторной решетке

В заключение описания метода неявного перебора по векторной решетке
необходимо отметить еще две его возможности, которые используются в
рассматриваемом здесь комбинированном методе \[2,22,63\].

Рассмотрим первую из них. Как видно из описания метода, при
самостоятельном его использовании перебор осуществляется в
гиперпараллелепипеде
$\left\{ 0 \div h\lbrack j\rbrack,(j = \overline{1,n}) \right\}$.
Назовем его исходным. Однако без какой-либо значительной перестройки
алгоритма перебор может быть организован в гиперпараллелепипеде
$\left\{ x_{\text{min}}\lbrack j\rbrack ¸h\lbrack j\rbrack,(j = \overline{1,n}) \right\}$,
который находится внутри полного за счет смещения в сторону увеличения
левых границ поиска по каждой оптимизационной переменной (назовем его
частичным). Очевидно, что в этом случае в сферу перебора попадает только
часть области допустимых решений исходной задачи, но в то же время
уменьшается и число перебираемых точек. Так как перебор в этом методе
идет в основном по недопустимым точкам, то процентное уменьшение числа
перебираемых точек является существенно более значительным, чем
уменьшение объема частичного гиперпараллелепипеда по отношению к объему
полного (Рис.2-4).

![](media/image6.wmf)

Рис.2‑4. Результаты работы метода неявного перебора при различных
$x_{\text{min}}$.

Вторая возможность состоит в том, что можно при переборе фиксировать
значения отдельных переменных. При этом перебор осуществляется не во
всем гиперпараллелепипеде (полном или частичном), а лишь в его сечениях.
Тогда можно постепенно наращивать пространство, в котором осуществляется
поиск, не повторяя ранее сделанный перебор. Так если ранее осуществлен
поиск в некотором частичном гиперпараллелепипеде
$\left\{ x_{\text{min}}\lbrack j\rbrack ¸h\lbrack j\rbrack,(j = \overline{1,n}) \right\}$,
то, выбрав некоторую оптимизационную переменную, у которой
$x_{\text{min}}\lbrack k\rbrack > 0$, можно зафиксировать ее значение на
уровне:

$x_{\text{min}}\lbrack k\rbrack: = x_{\text{min}}\lbrack k\rbrack - 1$,
(2.14)

и провести поиск в сечении:

$x\lbrack k\rbrack = x_{\text{min}}\lbrack k\rbrack,x\lbrack k\rbrack \in \left\{ x_{\text{min}}\lbrack j\rbrack \div h\lbrack j\rbrack(j = \overline{1,n},j \neq k) \right\}$.
(2.15)

В результате в совокупности перебор будет осуществлен в
гиперпараллелепипеде:

$\left\{ x_{\text{min}}\lbrack j\rbrack \div h\lbrack j\rbrack,(j = \overline{1,n}) \right\}$,
(2.16)

где $x_{\text{min}}\lbrack k\rbrack$ - новое значения, полученное в
соответствии c (2.14).

В частности, таким образом, размеры частичного гиперпараллелепипеда
можно постепенно довести до размеров полного. Эта возможность
используется на этапе 2 комбинированного метода при поиске первого
допустимого решения исходной задачи.

Обе указанные возможности метода неявного перебора по векторной решетке
определяют эффективность выполнения этапов 2 и 4. предлагаемого здесь
комбинированного метода.

### Преобразования исходной задачи для применения комбинированного метода

Использование в качестве основной части комбинированного метода
алгоритма неявного перебора по векторной решетке требует преобразование
задачи к следующему виду:

$Z_{\text{opt}}^{p} = \underset{x^{p}}{\text{min}}Z^{p} = c^{p}x^{p}\text{   }D^{p} =$
(2.17)

где:

- $x^{p}$ - $n$-мерный вектор-столбец основных оптимизационных
  переменных, соответствующих основным переменным исходной задачи;все
  $c^{p}\lbrack j\rbrack \geq 0\text{  }(j = 1,n)$, причем
  $c^{p}\lbrack 1\rbrack \leq c^{p}\lbrack 2\rbrack \leq \ldots \leq c^{p}\lbrack n\rbrack$;

- $h^{p}$ - $n$-мерный вектор-столбец ограничений сверху на
  оптимизационные переменные;

- все основные ограничения приведены к виду "$$"

Преобразование осуществляется путем реализации замены переменных по тем
оптимизационным переменным, при которых в исходной постановке (2.1)
коэффициенты целевой функции $c\lbrack j\rbrack < 0$:

$x\lbrack j\rbrack = h\lbrack j\rbrack - x^{p}\lbrack j\rbrack$, (2.18)

Затем для выполнения условия
$c^{p}\lbrack 1\rbrack \leq c^{p}\lbrack 2\rbrack \leq \ldots \leq c^{p}\lbrack n\rbrack$
осуществляется перенумерация оптимизационных переменных. Это
преобразование требуется не только для работы самого метода неявного
перебора, но и, как далее будет видно, для более простой алгоритмической
реализации второго этапа комбинированного метода. Вектор $h^{p}$
получается из вектора $h$ задачи (2.1) путем перестановки его компонент
в соответствии с перенумерацией переменных.

Преобразование основных ограничений к виду "$$" осуществляется путем
замены ограничений вида "=" двумя ограничениями противоположного вида:
"$$"и "$$". Затем все ограничения "$$" умножаются на "$- 1$". Это
преобразование необходимо лишь для применения метода неявного перебора.
В результате всех этих преобразований и получается матрица условий
$A^{p}$ размерности $m^{p} \times n$ и вектор правых ограничений $b^{p}$
размерности $m^{p}$. Обозначим $(Z_{\text{opt}}^{p},x_{\text{opt}}^{p})$
оптимальное решение задачи (2.17), если оно существует.

Дальнейшее изложение метода требует некоторых пояснений по
преобразованию исходной задачи для применения метода линейного
программирования. Здесь необходимо сказать, что предполагается применять
специальный алгоритм решения задач линейного программирования с
алгоритмическим учетом двусторонних ограничений на оптимизационные
переменные. Это определяется большими вычислительными преимуществами
этого варианта алгоритма симплекс-таблиц для рассматриваемой задачи.
Известно, что перед применением алгоритмов симплекс-таблиц ЗЛП должна
быть приведена к канонической форме. В данном случае эта форма будет
иметь следующий вид:

$Z_{\text{opt}}^{l} = \underset{x^{l}}{\text{max}}Z^{l} = c^{l}x^{l}\text{   }D^{l} =$,
(2.19)

$(Z_{\text{opt}}^{l},x_{\text{opt}}^{l})$ - решение

где:

$x^{l} = (x_{0}^{l},x_{d}^{l})^{T}$ - вектор-столбец размерности
$n^{l}$;

- $x_{0}^{l}$ - основные переменные задачи, полученные в результате
  преобразование исходной задачи к виду (2.17) (совпадают по нумерациями
  с переменными $x^{p}$ этой задачи);

- $x_{d}^{l}$ - дополнительные переменные, вводимые в ЗЛП для приведения
  ограничений к виду равенств;

- $c^{l} = ( - c^{p},0,\ldots,0)$ - вектор-строка коэффициентов целевой
  функции размерности $n^{l}$. Введение знака "--" перед $c^{p}$
  объясняется эквивалентным преобразованием задачи к направлению
  оптимизации на максимум, что требует каноническая форма ЗЛП. Таким
  образом, значения целевых функций задач (2.17) и (2.19) отличаются
  лишь знаками;

- $A^{l}$ - матрица условий ЗЛП размерности $m \times n^{l}$; Следует
  подчеркнуть, что матрица $A^{l}$ формируется из исходной $A$ в
  результате замены переменных (2.18) и введения дополнительных
  переменных. Преобразований ограничений к виду неравенств, как это
  делается при получении задачи (2.17), здесь не требуется;

- $b^{l}$ - вектор правых частей основных ограничений размерности $m$.
  Формируется на основе вектора $b$ с учетом замены переменных (2.18);

- $h^{l} = (h^{p},\infty,\ldots,\infty)^{T}$ - вектор-столбец
  размерности $n^{l}$ ограничений сверху на оптимизационные переменные.

Как видно, такое преобразование ЗЛП (2.19) обеспечивает ее простое
согласование по основным оптимизационным переменным и значению целевой
функции с задачей (2.17), что очень важно для реализации всего
комбинированного метода. Обозначим
$(Z_{\text{opt}}^{l},x_{\text{opt}}^{l})$ оптимальное решение задачи
(2.19), если оно существует.

### Совместное использование линейного программирования и неявного перебора по векторной решетке для нахождения допустимого субоптимального решения

Оптимальное решение задачи (2.17) есть целочисленная точка из $D^{l}$,
наиболее близкая по значению целевой функции к $Z_{\text{opt}}^{l}$.
Поэтому совершенно естественно осуществить поиск в ближайшей окрестности
точки $x_{\text{opt}}^{l}$, чтобы найти если и не оптимальное
$x_{\text{opt}}^{p}$, то хотя бы близкое к нему допустимое
субоптимальное решение. Это предполагается сделать путем ряда
вычислительных процедур. Вначале (этап 1 комбинированного метода)
решается задача (2.19). Если $D^{l} = \varnothing$, то аналогичный вывод
можно сделать и о допустимой области задачи (2.17)
($D^{p} = \varnothing$)и на этом прекратить дальнейший поиск. В
противном случае будет найдено оптимальное решение
$(Z_{\text{opt}}^{l},x_{\text{opt}}^{l})$. Случай неограниченности
целевой функции на $D^{l}$ здесь исключается, так как все
$h\lbrack j\rbrack$ и, следовательно, $h^{p}\lbrack j\rbrack$ имеют
фиксированные значения. Затем начинается реализация второго этапа метода
--- собственно поиск допустимого субоптимального решения. Этот этап
состоит из двух или трех подэтапов, в зависимости от результатов поиска.
Первый подэтап --- это поиск методом неявного перебора в
гиперпараллепипеде, определяемом $x_{\text{opt}}^{l}$. От точки
$x_{\text{opt}}^{l}$ в пространстве переменных $x^{p}$ делается как бы
шаг назад с использованием следующего соотношения:

$x_{\text{min}}^{p}\lbrack j\rbrack = \lbrack x_{\text{opt}}^{l}\lbrack j\rbrack\rbrack\text{  }(j = \overline{1,n})$,
(2.20)

где $\lbrack a\rbrack$ - целая часть числа $a$.

Этот шаг приводит в область недопустимых решения обеих задач (2.17) и
(2.19). Точку $x_{\text{min}}^{p}$ можно трактовать, как величину,
определяющую $n$--мерный гиперпараллелепипед:

$(x_{\text{min}}^{p}\lbrack j\rbrack \div h^{p}\lbrack j\rbrack,j = \overline{1,n})$,
(2.21)

в котором и предполагается начать поиска поиск допустимого решения
задачи (2.17) на втором этапе комбинированного метода (Рис.2-5).

![](media/image7.wmf)

Рис.2‑5. Результаты выполнения этапа 1 комбинированного метода.

Таким образом, должна решаться задача:

$\underset{x^{p}}{\text{min}}Z^{p} = c^{p}x^{p}\text{  }D_{\text{min}}^{p} =$,
(2.22)

Так как точка $x_{\text{min}}^{p}$ близка к границе допустимой области
$D^{l}$, то число перебираемых точек, как видно из предыдущего
рассмотрения, будет небольшим. Однако можно себе представить случай,
когда гиперпараллелепипед (2.21) не будет содержать ни одной допустимой
точки задачи (2.17). Один из них весьма экзотический, когда при непустом
множестве $D^{l}$ множество $D^{p} = \varnothing$ (Рис.2-6).

![](media/image8.wmf)

Рис.2‑6. Случай $D^{l} \neq 0$, а $D^{p} = 0$.

Возможны и другие случаи, определяемые своеобразным относительным
положением точки $x_{\text{min}}^{p}$ и области $D^{l}$ (Рис.2-7,
Рис.2-8).

В этих случаях необходимо осуществить второй подэтап поиска, постепенно
расширяя область поиска допустимого субоптимального решения. С
использованием возможностей метода неявного перебора по векторной
решетке это можно сделать эффективно, не повторяя поиск в уже
рассмотренных точках. Для этого необходимо рассматривать сечения
пространства $N^{p}$, расширяющие гиперпараллелепипед (2.21) в ту или
иную сторону, уменьшая на единицу и фиксируя значения отдельных
переменных. Таким образом, задачи (2.23) решаются до тех пор, пока либо
не будет найдено допустимое решение, либо не будет установлено, что
$D^{p} = \varnothing$:

$\underset{x^{p}}{\text{min}}Z^{p} = c^{p}x^{p}\text{  }D_{k}^{p} = \begin{array}{r}
\left\{ x^{p} \in N^{p}:A^{p}x^{p}b^{p}; \middle| \right\}
\end{array}\left\{ \right\}$,(2.23)

где:

- $k$ - номер переменной, в сторону уменьшения которой произошло на
  данном шаге расширения гиперпараллелепипеда (2.21);

- $x_{\text{min}}^{p}\lbrack k\rbrack$ - фиксированное значение $k$--ой
  переменной, при котором осуществляется поиск; определяется в
  соответствии со сказанным по отношению к значению
  $x_{\text{min}}^{p}\lbrack k\rbrack$ на предыдущем шаге расширения
  следующим образом:

$x_{\text{min}}^{p}\lbrack k\rbrack: = x_{\text{min}}^{p}\lbrack k\rbrack - 1$,
(2.24)

При выборе переменных, в сторону уменьшения которых целесообразно в
первую очередь расширять гиперпараллелепипед (2.21), целесообразно
руководствоваться правилом: в первую очередь расширять поиск в сторону
уменьшения переменной с наименьшим значением коэффициента целевой
функции (то есть переменной с наименьшим номером), так как область
допустимых решений в этом случае может расширятся от точки $D^{p}$ с
большей вероятностью именно в эту сторону (Рис.2-7).

![](media/image9.wmf)

Рис.2‑7. Одноэтапный вариант расширения области поиска субоптимального
решения.

Это является следствием переупорядочения переменных по возрастанию
$c^{p}\lbrack j\rbrack$. Последовательное расширение области поиска
осуществляется, как уже говорилось, итеративно до нахождения первого
допустимого решения. При таком расширении точка $x_{\text{min}}^{p}$
постепенно смещается в сторону начала координат. Очевидно, что
расширение гиперпараллелепипеда (2.21) в сторону уменьшения переменных,
для которых $x_{{}^{\text{min}}}^{p}\lbrack j\rbrack = 0$ невозможно. Из
дальнейшего рассмотрения при расширении гиперпараллелепипеда могут быть
также исключены те переменные, при расширении в сторону которых при
решении задачи (2.23) в начальной точке поиска сработал критерий КН
метода перебора по векторной решетке. Это значит, что в этом и следующих
сечениях пространства $N^{p}$ по этой переменной никогда не будет
найдено их пересечение с областью допустимых решений $D^{p}$ (Рис.2-8).

![](media/image10.wmf)

Рис.2‑8. Двухэтапный вариант расширения области поиска субоптимального
решения.

На этом рисунке видно, что первое же расширение гиперпараллелепипеда в
сторону уменьшения переменной
$x^{p}\lbrack 1\rbrack:(x_{\text{min}}^{p}\lbrack 1\rbrack = 1)$
приводит к такой ситуации. Следовательно, нужно перейти к рассмотрению
сечений по другой переменной $x^{p}\lbrack 2\rbrack$.

Если при таком расширяющемся поиске после рассмотрения всех возможных
сечений не найдено допустимого решения задачи (2.17), то это означает,
что область допустимых решений $D^{p} = \varnothing$. Таким достаточно
сложным в вычислительном отношении способом устанавливается при
применении рассматриваемого комбинационного метода случай:

$(D^{l} \neq \varnothing) \cap (D^{p} = \varnothing)$, (2.25)

К счастью, для реальных задач типа (2.1) это, как уже говорилось, -
редкий случай. Наиболее вероятно возникновение такой ситуации для задач
с основными ограничениями в виде равенств.

При успешном поиске допустимого решения задачи (2.17), обозначим его
$(z_{D^{p}}x_{D^{p}})$, сразу (на подэтапе 1) или после дополнительного
расширения области поиска (подэтап 2) целесообразно рассмотреть простые
возможности его улучшения (подэтап 3). Улучшенное допустимое решение
обозначим $({\overset{\sim}{z}}_{D^{p}}{\overset{\sim}{x}}_{D^{p}})$.
Чем лучше будет найдено допустимое решение задачи (2.17) на этом этапе,
тем меньше вычислений необходимо будет сделать на следующих двух этапах
метода. Ситуация, когда такое улучшение возможно простыми средствами,
показана на Рис.2-9.

![](media/image11.wmf)

Рис.2‑9. Случай, когда улучшение возможно простым способом.

Правила, реализующие простые возможности улучшения, основаны на
преобразовании задачи к виду (2.17), когда все
$c^{p}\lbrack j\rbrack \geq 0$. Суть их следующая. Осуществляется поиск
переменных, по которым возможно простое улучшение, т.е. переменных,
которым соответствуют $c^{p}\lbrack j\rbrack \geq 0$, и уменьшение
которых возможно без выхода из области допустимых решений $D^{p}$. Из
всех таких переменных выбирается та, которая обеспечивает наибольшее
уменьшение целевой функции задачи (2.17). Это улучшение определяется как
величиной $c^{p}\lbrack j\rbrack$, так и величиной возможного уменьшения
значения той оптимизационной переменной
${\Delta x}^{p}\lbrack j\rbrack = x_{D^{p}}\lbrack j\rbrack - {\overset{\sim}{x}}_{D^{p}}\lbrack j\rbrack$,
по которой возможно простое улучшение. Если простое улучшение решения
$({\overset{\sim}{z}}_{D^{p}}{\overset{\sim}{x}}_{D^{p}})$, найденного в
результате первого или второго подэтапов, невозможно, то будем считать,
что улучшенное решение совпадает с найденным допустимым:

$({\overset{\sim}{z}}_{D^{p}},{\overset{\sim}{x}}_{D^{p}}) \equiv (z_{D^{p}},x_{D^{p}})$,
(2.26)

Следует подчеркнуть, что отсутствие простых возможностей улучшения
решения или их исчерпание не означает, что дальнейшее улучшение вообще
не возможно (Рис. 2-10).

### Совместное использование линейного программирования и метода неявного перебора по векторной решетке для окончательного поиска оптимального решения

Полученное на предыдущем этапе допустимое субоптимальное решение
$({\overset{\sim}{z}}_{D^{p}}{\overset{\sim}{x}}_{D^{p}})$ задачи (2.17)
позволяет сформировать так называемое фильтрующее ограничение. Это
понятие широко используется в методах неявного перебора. В методе
неявного перебора по векторной решетке на нем основано действие критерия
КПИА. Там оно имеет вид строгого неравенства:

$c^{p}x^{p} \leq {\overset{\sim}{z}}_{D^{p}}$ (2.29)

Суть этого неравенства в том, что после нахождения допустимого решения
${\overset{\sim}{x}}_{D^{p}}$ co значением целевой функции
${\overset{\sim}{z}}_{D^{p}}$ в дальнейшем ищутся только решения,
которые лучше найденного.

Здесь предлагается помимо обычного способа применения фильтрующего
ограничения использовать его в ЗЛП (2.4) в виде:

$c^{p}x^{l_{0}} \leq {\overset{\sim}{z}}_{D^{p}}$, (2.30)

В отличие от (2.29) в (2.30) участвует другое обозначение тех же самых
по номерам оптимизационных переменных (они в ЗЛП в отличие от
целочисленной задачи принимают действительные значения), а также само
неравенство имеет нестрогий вид ("$$"), так как симплекс - методы не
работают со строгими неравенствами. Непосредственная цель введения
фильтрующего ограничения (2.30) в ЗЛП - выделить из области допустимых
решений $D^{l}$ подобласть:

$D_{f}^{l} = D^{l} \cap (c^{p}x_{0}^{l} \leq {\overset{\sim}{z}}_{D^{p}})$,
(2.31)

которая содержит все решения $x^{p}$, лучшие чем
${\overset{\sim}{x}}_{D^{p}}$, если они существуют. В частном случае их
может и не быть, если найденное субоптимальное решение
${\overset{\sim}{x}}_{D^{p}}$ в действительности является оптимальным
решением задачи (2.17). Только утверждать это по итогам выполнения двух
первых этапов нет оснований.

Конечная же цель введения фильтрующего ограничения в ЗЛП (2.19) -
определить квадрант, включающий в себя все целочисленные решения задачи
(2.17), содержащиеся в области $D_{f}^{l}$, и начальная точка
$x_{\text{min}}$ которого максимально приближена к границам этой области
(Рис.2-11).

![](media/image13.wmf)

Рис.2‑11. Выполнение всех четырех этапов комбинированного метода.

Поиск в этом квадранте с его начальной точки методом неявного перебора
по векторной решетке, с одной стороны, гарантирует нахождение
оптимального решения задачи (2.31), а, с другой стороны, достаточная
близость начальной точки этого квадранта к системе основных ограничений
обеспечивает минимальную вычислительную сложность процесса поиска
оптимального решения. Определение указанной начальной точки
$x_{\text{min}}$ может быть сведено к решению ряда задач по нахождению
минимальных значений координат, соответствующих границам области
$D_{f}^{l}$ в пространстве $N^{p}$:

$Z_{j\text{opt}}^{l} = \underset{x^{l} \in D_{f}^{l}}{\text{min}}Z_{j}^{l} = c_{j}^{l}x^{l} = x^{l}\lbrack j\rbrack$,
(2.32)

или, что одно и тоже,

$\underset{x^{l} \in D_{f}^{l}}{\text{max}} - Z_{j}^{l} = - x^{l}\lbrack j\rbrack$,
(2.33)

с последующим пересчетом:

$x_{\text{min}}\lbrack j\rbrack = \rbrack z_{j\text{opt}}^{l}\lbrack$,
(2.34)

где $\rbrack a\lbrack$ - наименьшее целое, но не меньше $a$.

Вычислительная операция (2.34) позволяет перейти от минимальных
действительных к минимальным целым значениям границ.

Перечень решаемых задач (2.33) определяется множеством:

$J = \left\{ j \in \overline{1,n}:\text{  }N(B_{\text{opt}}^{l}) \cup N_{\text{opt}}^{l} \right\}$,
(2.35)

где:

- $N(B_{\text{opt}}^{l})$ - базисное множество для оптимального решения
  задачи (2.19);

- $N_{\text{opt}}^{l}$ - множество переменных, вышедших в оптимальном
  решении задачи (2.19) на свои верхние ограничения. Это множество
  является атрибутом указанного выше специального метода симплекс-таблиц
  для задач с двусторонними ограничениями на переменные, применяемого
  здесь.

Условие (2.35) обосновывается следующим рассуждением. Так как
$x_{\text{opt}}^{l}$ представляет собой одну из крайних точек
$D_{f}^{l}$, то нулевые небазисные значения отдельных компонент
$x_{\text{opt}}^{l}\lbrack j\rbrack$ в этой точке и являются их
минимальными значениями. Следовательно, решать задачи (2.33) нужно лишь
для тех $j \in \left( \overline{1,n} \right)$, которые принадлежат либо
множеству $N(B_{\text{opt}})$, либо - $N_{\text{opt}}^{}$.

Относительно характеристики и процесса решения задач (2.33) необходимо
сделать следующие важные замечания.

Задачи (2.33) являются задачами линейного программирования,
отличающимися от ЗЛП (2.19) целевой функцией (здесь
$c\lbrack j\rbrack = - 1$ остальные коэффициенты целевой функции
нулевые) и одним дополнительным (фильтрующим) ограничением (2.30),
которое не отсекает крайнюю точку с оптимальным решением
$x_{\text{opt}}^{l}$ Это и определяет эффективную возможность решения
каждой такой задачи не с самого начала, а с крайней точки,
соответствующей найденному оптимальному решению задачи (2.19). Так как
минимальные значения оптимизационных переменных соответствуют крайним
точкам области $D_{f}^{l}$, как правило, достаточно близко расположенным
по отношению к точке прежнего оптимального решения, если не соседним с
ней по исходящим из нее ребрам (это определяется субоптимальностью
найденного на предыдущем этапе решения
$({\overset{\sim}{z}}_{D^{p}}{\overset{\sim}{x}}_{D^{p}})$, в
соответствии с которым сформировано фильтрующее ограничение), то число
итераций симплекс - метода будет соответственно небольшим: иногда даже
нулевым, если оптимальное решение задачи (2.33) соответствует точке
$x_{\text{opt}}^{l}$, а чаще - равным единице, если искомое решение
является соседним с $x_{\text{opt}}^{l}$ по ребру допустимой
области$D_{f}^{l}$. Для того, чтобы осуществить такой подход, необходимо
корректировать структуру исходных данных задачи (2.19) с учетом
указанных выше отличий от нее задачи (2.33), а самое главное - уметь
корректировать структуру данных, определяющих прежнее оптимальное
решение $x_{\text{opt}}^{l}$ (то есть заключительную симплекс - таблицу
задачи (2.19), вводя в нее фильтрующее ограничение и пересчитывая ее
элементы, определяемые новыми коэффициентами целевой функции задачи
(2.33)). Как это сделать, будет определено ниже при формальном описании
алгоритма комбинированного метода.

И, наконец, определим соображения по осуществлению заключительного
(четвертого) этапа комбинированного метода по нахождению оптимального
решения задачи (2.17). Чаще всего он будет сводится к решению методом
неявного перебора по векторной решетке задачи, аналогичной задаче (2.22)
со следующими отличиями: вместо $x_{\text{min}}^{p}$ необходимо
использовать вектор $x_{\text{min}}$, определенный в результате
выполнения предыдущего этапа комбинированного метода и, кроме того,
сразу же, с первых шагов, должно быть включено в работу фильтрующее
ограничение в виде строгого неравенства (2.29), правая часть которого
затем должна корректироваться по общим правилам метода неявного
перебора. Определим эту задачу следующим образом:

$\underset{x^{p}}{\text{min}}Z^{p} = c^{p}x^{p}\text{   }D_{\text{min}} =$
(2.36)

Действие фильтрующего ограничения исключит из рассмотрения точку
субоптимального решения ${\overset{\sim}{x}}_{D^{p}}$ и эквивалентные ей
по значению целевой функции точки из области $D_{f}^{l}$, Таким образом,
может оказаться (а часто это так и будет), что по результатам вычислений
на четвертом этапе комбинированного метода будет делаться вывод о том,
что найденное на втором этапе субоптимальное решение и является
оптимальным решением задачи (2.17), когда
$D_{\text{min}} = \varnothing$. Иногда для такого вывода и не нужно
решать задачу (2.36). Это случай, когда окажется, что
$x_{\text{min}} \leq x_{\text{min}}^{p}$ то есть область допустимых
решений задачи (2.36) является подобластью области допустимых решений
задачи (2.22) $D_{\text{min}} \subset D_{\text{min}}^{p}$ (конечно, с
учетом возможного ее расширения на подэтапе 2 этапа 2 комбинированного
метода). Если же оптимальное решение задачи (2.36) найдено
$(z_{D_{\text{min}}}x_{D_{\text{min}}})$, то оно и будет оптимальным
решением задачи (2.17).

В заключение работы метода, если будет найдено оптимальное решение
задачи (2.17), останется осуществить переход к оптимальному решению
исходной задачи (2.1) по правилам, обратным к правилам преобразования
задачи (2.1) в задачу (2.17). Такое же преобразование необходимо сделать
и в случае останова работы метода по допустимой погрешности на этапе 2.

## Алгоритм комбинированного метода

1.  Преобразование исходной задачи к виду (2.17). Формирование исходных
    данных ЗЛП (2.19).

2.  Решение ЗЛП (2.19) с помощью специального алгоритма симплекс-таблиц
    для ЗЛП с двусторонними ограничениями на переменные. Заключительную
    симплекс-таблицу сохранить до окончания выполнения п.3. Выводы по
    результатам решения:

Если $D^{l} = \varnothing$, то$D^{p} = \varnothing$ и $D = \varnothing$,
переход на 6;

Если $x_{\text{opt}}^{l} \in D^{p}$, то переход на 5;

Если $x_{\text{opt}}^{l} \notin D^{p}$, то определяется
$x_{\text{min}}^{p}$(2.20).

3.  Поиск субоптимального решения задачи (2.17)

    1.  Поиск начального допустимого решения задачи (2.17)

        1.  Решение задачи (2.22) методом неявного перебора по векторной
            решетке.

        2.  Если оптимальное решение задачи (2.22) или (2.23) найдено,
            то переход на 2.2.

        3.  Если сформировать дополнительную область поиска допустимого
            субоптимального решения невозможно, то $D^{p} = \varnothing$
            и $D = \varnothing$, переход на 6.

        4.  Выбор переменной, по которой осуществляется формирование
            дополнительной области поиска допустимого субоптимального
            решения. Корректировка $x_{\text{min}}^{p}$ с учетом
            расширения области поиска (2.25).

        5.  Формирование и решение задачи (2.23). Переход на 2.1.2

    2.  Поиск и реализация возможностей улучшения начального допустимого
        решения задачи (2.17).

        1.  Если переменные, по которым возможна простая стратегия
            улучшения найденного начального допустимого решения задачи
            (2.17), отсутствуют, то переход на 2.3.

        2.  Определение перечня переменных, по которым возможно
            улучшение.

        3.  Выбор переменной, которая обеспечивает наибольшее уменьшение
            целевой функции задачи (2.17). Переход на 2.2.1.

    3.  Если найденное решение удовлетворяет требованиям по допустимой
        погрешности, то переход на 5.

4.  Определение начальной координаты квадранта $x_{\text{min}}$ для
    окончательного поиска.

    1.  Введение фильтрующего ограничения в структуру данных и
        заключительную симплекс-таблицу задачи (2.19).

    2.  Для всех $j \in J$ (2.35) выполнить:

        1.  Формирование структуры данных и начальной симплекс --
            таблицы для решения ЗЛП (2.33).

        2.  Решение ЗЛП (2.33) и определение
            $x_{\text{min}}\lbrack j\rbrack$ (2.34).

5.  Окончательный поиск оптимального решения

    1.  Если $x_{\text{min}} \leq x_{\text{min}}^{p}$, то переход на 5.

    2.  Решение задачи (2.36) методом неявного перебора по векторной
        решетке. Если $D_{\text{min}} = \varnothing$, то
        $x_{\text{opt}}^{p} = {\overset{\sim}{x}}_{D^{p}}$

6.  Переход к оптимальному или субоптимальному решению исходной задачи.

7.  Конец.

## Использование эвристических алгоритмов для улучшения свойств комбинированного метода

Иногда при решении оптимизационных задач имеет смысл использовать
некоторую априорную информацию о решаемой задаче, если она есть, и если
это позволить ускорить процедуру нахождения оптимального решения. В этом
случае появляются так называемые эвристические алгоритмы, и существующие
оптимизационные могут получать некоторые модификации, связанные с
использованием эвристик.

Применение эвристических алгоритмов может приводить к тому, что
оптимизационный метод теряет свою универсальность, что не является
положительным моментом. При решении задачи по определению оптимальной
топологии ЛВС будет использована эвристика, которая позволит значительно
улучшить свойства комбинированного метода.

Одним из примеров использования эвристики в решении оптимизационных
задач может являться специальный алгоритм определения допустимого
решения транспортной задачи -- метод минимального элемента. Несмотря на
то, что транспортная задача относится к более широкому классу задач
линейного программирования, чем целочисленные задачи, ее рассмотрение с
точки зрения применения эвристик является хорошим примером.

В этом алгоритме используются некоторые идеи, например:

- способ заполнения матрицы при процедуре пересчета, когда берется
  минимальное значение двух незаполненных перевозок;

- тот факт, что порядок заполнения элементов в матрице жестко не задан,

которые не применимы для всех задач этого класса, а применяются лишь при
решении транспортной задачи, так как используют конкретные свойства
именно этой задачи. Но использование этого алгоритма в решении
транспортной задачи гораздо выгоднее, чем применение общего подхода для
поиска допустимого решения, так значительно быстрее и проще дает
требуемый результат.

В комбинированном методе явным местом, где может применяться эвристика,
является нахождение допустимого решения на втором этапе. Здесь эвристика
может заменить применение метода неявного перебора по векторной решетке.

## Свойства комбинированного метода

Комбинированный метод основан на комбинации возможностей линейного
программирования, неявного перебора и применения эвристик, если это
позволяет решаемая задача. Идея его состоит в последовательном
проведении следующих четырех этапов вычислений. На первом этапе решается
задача линейного программирования (ЗЛП), полученная из основной путем
снятия условий целочисленности на переменные. Полученное оптимальное
решение в случае его нецелочисленности становится отправной точкой для
второго этапа реализации метода: поиска методом неявного перебора
допустимого субоптимального решения исходной задачи в окрестности
найденного на первом этапе нецелочисленного решения. На данном этапе
можно применить эвристические алгоритмы более быстрого поиска
допустимого решения, чем применение метода неявного перебора по
векторной решетке, если это позволяют условия задачи. Третий этап
состоит в формировании на основе целевой функции исходной задачи и
найденного субоптимального решения так называемого фильтрующего
ограничения, отсекающего от области допустимых решений ЗЛП большую часть
с худшими решениями, чем найденное субоптимальное. Это ограничение
вводится в структуру данных, сформированной и решенной на первом этапе
ЗЛП. С использованием возможностей линейного программирования на этом же
этапе определяются границы, отсеченной фильтрующим ограничением
подобласти допустимых решений исходной задачи, которая содержит все
лучшие решения, чем найденное субоптимальное (если, конечно, они
существуют). Сформированная таким образом подобласть с установленными ее
границами является основой для определения малой области для
окончательного поиска оптимального решения исходной задачи методом
неявного перебора на заключительном четвертом этапе. Таким образом,
гарантируется нахождение оптимального решения исходной задачи, если оно
существует. Но имеется возможность остановится и на найденном
субоптимальном решении, оценив сверху возможную его погрешность по
значению целевой функции. Использование метода неявного перебора на
втором и четвертом этапах поиска не должно приводить к большой
вычислительной сложности, так как гарантируется малый размер областей
поиска в обоих случаях. Достоинством метода по сравнению с методами
правильных отсечений и ветвей и границ является исключение влияния
накопления ошибок при решении ЗЛП на надежность поиска. Априорный анализ
показывает, что преимущества предлагаемого комбинированного метода будут
возрастать при увеличении размерности решаемой задачи. Кроме этого, в
комбинированном методе есть возможность применения различных
эвристических алгоритмов для более точной локализации области перебора
после первого этапа метода.

## Программная реализация комбинированного метода

В данном разделе приводится описание программной реализации
комбинированного метода решения целочисленных линейных оптимизационных
задач (далее по тексту -- комбинированного метода) \[25,38\].

### Язык программирования

Кроме концептуальных и алгоритмических особенностей, которые придали
комбинированному методу его отличительные свойства, немаловажную роль
сыграло качество его программной реализации. В задачах большой
размерности очень сложно вручную проверить результат работы того или
иного метода, поэтому на первый план выходит качество самого
программирования. Программирование с использованием
объектно-ориентированного анализа и проектирования позволяет избежать
большинства потенциальных ошибок уже на стадии создания программы. На
отладку и тестирование остается выявление не мелких ошибок, появившихся
в процессе написания программы, а проверка глобальных идей, заложенных в
алгоритме комбинированного метода. Программная метода была выполнена на
языке C++ \[9,47,49\]. Созданная иерархия классов позволила разделить
алгоритм на части, которые программировались и отлаживались независимо
друг от друга. Являясь комбинированным методом, предлагаемый метод
использует в себе несколько классических оптимизационных методов,
например, симплекс-метод, метод неявного перебора по векторной решетке.
При данном подходе, все используемые методы можно легко выносить в
самостоятельные программы для тех или иных экспериментов. При
необходимости расширения функциональности того или иного модуля
сокращена к минимуму опасность нарушить работоспособность других
модулей. Кроме того, в дальнейшем, при развитии метода или создания на
его базе нового, будет возможность использовать уже созданные классы как
базу для новой программной реализации. Синтаксис же языка C++ позволил
написать программный код надежным и в тоже время ясным для дальнейшего
сопровождения. Одной из проблем в программировании подобных задач (когда
используются большие объемы памяти для хранения данных) является
контроль использования динамической памяти. Механизмы автоматического
создания и уничтожения объектов в языке С++ максимально облегчили эту
задачу. Язык С++, являясь языком программирования, в полной мере
реализующего парадигму объектно-ориентированного анализа и
проектирования, позволил создать программную реализацию комбинированного
метода, отвечающую стандартам современного программного обеспечения.

### Особенности реализации метода ветвей и границ с использованием специальных алгоритмов решения ЗЛП с двухсторонними ограничениями на оптимизационные переменные

Применяется метод ветвей и границ с процедурой ветвления множества $D'$,
полученном из $D^{n}$ путем снятия условия целочисленности на
оптимизационные переменные.

#### Постановка задачи оптимизации

$\underset{x^{n}}{\text{max}}c^{n}x^{n}\text{   }D^{n} =$, (3.1)

где $h^{n}$ - целочисленный вектор правых частей ограничений.

Основная часть алгоритма работает с задачей, приведенной к канонической
форме:

$\underset{x^{k}}{\text{max}}c^{k}x^{k}\text{   }D^{k} =$, (3.2)

Для первоначального решения ЗЛП на $D'$ используется специальный

Для решения ЗЛП, полученной в результате ветвления $D'$, используется
разработанный для этих целей специальный алгоритм двойственного
симплекс-метода \[26\] с алгоритмическим учетом двусторонних
ограничений. Структура данных используемая этим методом такая, как и в
прямом специальном алгоритме поиска оптимального решения.

#### Специальный алгоритм двойственного симплекс-метода для задач с двусторонними ограничениями на оптимизационные переменные.

Данный алгоритм разработан для реализации двойственных вариантов МВГ и
МПО, в которых он используется для нахождения нового оптимального
решения каждый раз либо после очередного ветвления (в МВГ) либо после
очередного правильного отсечения (в МПО).

Предполагается, что задача приведена к виду:

$\text{maxcx   }D = \left\{ x \in R^{n}:\text{Ax} = b\text{  }0 \leq x \leq h \middle| \middle| x \right\}$
с числом основных ограничений $m$.

Алгоритм начинает работать с сопряженного разбиения переменных на
подмножества $N(B_{1})$, $N_{1}^{}$, $N_{1}^{0}$, при котором все
симплекс-разности для $j \in N_{1}^{}$ неотрицательны и все
симплекс-разности для $j \in N_{1}^{0}$ неположительны.

Алгоритм работает со структурой данных почти полностью аналогичной
структуре данных, используемых в прямом специальном алгоритме решения
ЗЛП с двусторонними ограничениями на переменные.

В отличие от прямого алгоритма по структуре данных в двойственном помимо
симплекс-разностей должны генерироваться и разрешающие строки матрицы
$\overline{(A)}(B_{k})$.

1.  $$k: = 1$$

2.  Если условие оптимальности выполняется, то есть:\
    $0 \leq b(B_{k},N_{k}^{})\lbrack i\rbrack \leq h\lbrack j_{i}\rbrack$
    для любого $i \in (1,m)$, то выписать оптимальное решение, переход
    на п.10.

3.  Определить номер разрешающей строки $r$:

    1.  $\Delta_{\text{max}} = \Delta_{i}$, где
        $\Delta_{i} = \begin{array}{r}
        \left\{ \left| b(B_{k},N_{k}^{})\lbrack i\rbrack \right|,\text{ если }b(B_{k},N_{k}^{})\lbrack i\rbrack < 0 \middle| \right.\ 
        \end{array}$

    2.  $$r = \text{min}\left\{ i:\Delta_{i} = \Delta_{\text{max}} \right\}$$

    3.  Зафиксировать, какой из двух случаев произошел:

        1.  $r = r_{1}$, если $b(B_{k},N_{k}^{})\lbrack r\rbrack < 0$

        2.  $r = r_{2}$, если
            $b(B_{k},N_{k}^{})\lbrack r\rbrack > h\lbrack j_{r}\rbrack$

4.  Генерировать $r$-ю строку матрицы $\overline{(A)}(B_{k})$:
    ${\overline{a}}_{r}(B_{k}) = {\overline{d}}_{r}\overline{A}$

    Если:\
    ($(r = r_{1}) \cap (\overline{a}(B_{k})\lbrack\text{rj}\rbrack \geq 0\text{  }\forall j \in N_{k}^{0})$ $$ $(\overline{a}(B_{k})\lbrack\text{rj}\rbrack \leq 0\text{  }\forall j \in N_{k}^{})$) $$\

    то переход $D = \varnothing$, переход на10.

7.  Определить номер разрешающего столбца $l$:

    1.  Если $r = r_{1}$, то множество $J_{B}$ определяет следующим
        образом:\
        $J_{B} = \left\{ j \in \overline{1,n}:(\overline{a}(B_{k})\lbrack\text{rj}\rbrack < 0\text{   }\forall j \in N_{k}^{0}) \cup (\overline{a}(B_{k})\lbrack\text{rj}\rbrack \geq 0\text{   }\forall j \in N_{k}^{}) \right\}$\

#### Проведение ветвлений без увеличения размерности задачи по числу ограничений и оптимизационных переменных

Пусть первоначально решена задача, полученная из задачи (3.2) путем
снятия условия целочисленности на оптимизационные переменные:

$\underset{x \in D'}{\text{max}}\text{cx}$, (3.3)

и получено оптимальное решение: $x_{D'}^{0} \notin D^{n}$.

Пусть $x^{0}\lbrack j\rbrack$ - первая из компонент вектора
$x_{D'}^{0}$, не являющаяся целочисленной. Ветвление множества $D'$
осуществляется по этим переменным ниже описанным образом.

![](media/image16.wmf)

Рис.3‑14. Процедура ветвления множества $D'$.

$D_{1}' = D' \cap (x\lbrack j\rbrack \leq \lbrack x^{0}\lbrack j\rbrack\rbrack)$,
(3.4)

$D_{2}' = D' \cap (x\lbrack j\rbrack \leq \lbrack x^{0}\lbrack j\rbrack\rbrack + 1)$,
(3.5)

Таким образом, в результате ветвления возникает две задачи:

$\underset{x \in D_{1}'}{\text{max}\text{cx}}$ (3.6)

$\underset{x \in D_{2}'}{\text{max}\text{cx}}$ (3.7)

которые от задачи (3.3) отличаются введением дополнительных ограничений
типа неравенств в соответствии с соотношениями (3.4) и (3.5). При
формальном подходе их необходимо было бы преобразовать к канонической
форме и ввести в структуру основных ограничений решаемой задачи. При
этом увеличилось бы число ограничений и переменных задачи. Так как число
ветвлений, в результате которых на концах ветвей образуются подмножества
$D_{i}'$ будет в различных задачах неодинаковым и заранее неизвестным,
то при таком подходе возникли бы серьезные проблемы с хранением данных
по каждой из образующихся при ветвлении задач. Покажем, как эту проблему
можно устранить.

Проще всего без увеличения ограничений и оптимизационных переменных
осуществляется формирование задачи (3.6). Для этого в задаче (3.3)
необходимо лишь изменить ограничения сверху на оптимизационную
переменную $x\lbrack j\rbrack$:

$h_{D_{1}'} = \lbrack x^{0}\lbrack j\rbrack\rbrack$, (3.8)

Несколько сложнее обстоит дело с задачей (3.7). Без увеличения числа
ограничений и оптимизационных переменных нельзя получить задачу
полностью эквивалентную задаче (3.7). Однако путем смещения начала
координат по $j$-й переменной в сторону положительных значений на
величину $\lbrack x^{0}\lbrack j\rbrack\rbrack + 1$:

$x\lbrack j\rbrack: = x\lbrack j\rbrack - (\lbrack x^{0}\lbrack j\rbrack\rbrack + 1)$,
(3.9)

можно получить задачу квазиэквивалентную задаче (3.7).

При этом конфигурация области допустимых решений будет точно такой же,
как и в задаче (3.7), а от оптимальных решений скорректированной задачи
можно перейти к оптимальному решению задачи (3.7), запомнив сделанное
выше смещение:

$x_{D_{2}'}^{0} = x^{0}\lbrack j\rbrack + x^{s}\lbrack j\rbrack$, (3.10)

где:

$x^{0}\lbrack j\rbrack$ - оптимальное значение $j$-й переменной
скорректированной задачи;

$x^{s}\lbrack j\rbrack: = (x^{0}\lbrack j\rbrack\rbrack + 1)$ - смещение
по $j$-й переменной.

Формально получение квазиэквивалентной задачи из задачи (3.3) может быть
осуществлено путем замены переменных:

$x\lbrack j\rbrack = x'\lbrack j\rbrack + x^{s}\lbrack j\rbrack$, (3.11)

где $x\lbrack j\rbrack$, $x'\lbrack j\rbrack$ - соответственно старое и
новое значение $j$-й оптимизационной переменной.

При такой замене целевая функция оптимизационной задачи не изменится
(образующуюся константу можно отбросить, она не влияет на поиск
оптимального решения).

Изменения лишь коснутся векторов $b$ и $h$:

$b_{D_{2}^{S'}} = b_{D'} - x^{s}\lbrack j\rbrack a^{j}$, (3.12)

где $a^{j}$ - $j$-й вектор-столбец матрицы $A_{k}$

$h_{D_{2}^{S'}} = h_{D'} - x^{s}\lbrack j\rbrack$, (3.13)

то есть вектор $b$ изменяется в общем случае весь, а в векторе $h$
изменяется лишь $j$-я компонента.

В дальнейшем, если возникает необходимость продолжить ветвление по ветви
с множеством $D_{2}$, для сохранения числа ограничений и оптимизационных
переменных необходимо продолжить работать со смещенным множеством
$D_{2}^{s}$, проведя изменения аналогичные указанным выше и суммируя
получающиеся смещения с теми, которые сделаны ранее для множества,
подвергающегося ветвлению. Таким образом, в результате многократного
применения процедур ветвления на конце каждой ветви будет формироваться
ЗЛП:

$\underset{x \in D_{i}^{S'}}{\text{max}\text{cx}}$, (3.14)

у которой по отношению к задаче (3.3) будет свой вектор ограничений
сверху на оптимизационные переменные $h_{D_{i}^{S'}}$, свой вектор
правых частей ограничений $b_{D_{i}^{S'}}$, свое смещение
$x_{D_{i}^{S'}}^{S}$. Эти значения аккумулируют все изменения,
произошедшие в результате ветвлений.

Общие соотношения, аналогичные соотношениям (3.8, 3.12, 3.13), в
соответствии с которыми будут изменяться параметры задач, образующихся в
результате дальнейшего ветвления множества $D_{i}^{S'}$ будут иметь вид:

$h_{D_{i1}^{S}} = \lbrack x^{0}\lbrack j\rbrack\rbrack$, (3.15)

$b_{D_{i2}^{S'}} = b_{D_{i}'} - x^{s}\lbrack j\rbrack a^{j}$, (3.16)

$h_{D_{i2}^{S'}} = h_{D_{i}'} - x^{s}\lbrack j\rbrack$, (3.17)

Формулы для смещений подмножеств $D_{i1}^{S'}$ и $D_{i2}^{S'}$ будут
иметь следующий вид:

$x_{D_{i1}^{S'}}^{S} = x_{D_{i}^{S'}}^{S}$, (3.18)

$x_{D_{i2}}^{S} = x_{D_{i}} + \begin{array}{r}
\left\lbrack 0 \middle| \right\rbrack\left\lbrack \vdots \middle| \right\rbrack\left\lbrack 0 \middle| \right\rbrack\left\lbrack (x^{S})_{j} \middle| \right\rbrack\left\lbrack 0 \middle| \right\rbrack\left\lbrack \vdots \middle| \right\rbrack \\

#### Структура данных, используемая в алгоритмической реализации, и правила работы с ней

Блок данных по каждой задаче, образовавшейся на конце той или иной
ветви, имеет следующий состав:

$b_{D_{i}^{S'}}$, $h_{D_{i}^{S'}}$, $x_{D_{i}}^{S}$,
$N(B_{\text{opt}})$, $N_{\text{opt}}^{}$, $N_{\text{opt}}^{0}$,
${\overline{B}}^{- 1}$,
$\overline{b}(B_{\text{opt}},N_{\text{opt}}^{})$,

$\xi_{D_{i}} = ( - D_{i\text{opt}}^{S})$ - оценка снизу целевой функции
(с учетом перехода к исходным координатам и направлению оптимизации на
максимум).

При этом необходимо иметь в виду, что расширенная матрица условий
${\overline{A}}^{k} = \left( \frac{A^{k}}{c^{k}} \right)$ и размерности
по числу ограничений $m^{k}$ и переменных $n^{k}$ одинаковы для всех
образующихся при ветвлении задач.

Для хранения совокупности данных по всем ЗЛП на концах ветвей
используется списковая структура, элементом которой является блок
данных, описанный выше.

Элементы списка упорядочены по возрастанию $\xi_{D_{i}}$.

Правила работы со списковой структурой определены следующим образом:

- первоначально в список заносятся результаты решения ЗЛП на $D'$;

- ветвление осуществляется по задаче, блок данных которой стоит на
  первом месте в списке;

- после формирования и решения ЗЛП на подмножествах $D_{i1}^{S'}$ и
  $D_{i2}^{S'}$ блок данных, соответствующий задаче $D_{i}'$ должен быть
  из списка удален;

- блок данных, соответствующий задаче, образовавшейся в результате
  ветвления и у которой область допустимых решений пуста, в список не
  заносится;

- блок данных, соответствующий задаче, у которой $x^{0} \in D^{n}$, в
  список не заносится;

- блок данных по задаче, у которой $\xi_{D_{i}} \geq \eta(D)$, в список
  не заносится;

- после очередной корректировки $\eta(D)$ хвост списковой структуры,
  соответствующий блокам с $\xi_{D_{i}} \geq \eta(D)$, отсекается.

Правила формирования исходных данных для решения специальным
двойственного симплекс-метода задач, образующихся в результате ветвления
множества $D_{i}^{S'}$ имеет следующий вид:

При формировании данных по задаче $D_{i1}^{S'}$:

- осуществляется корректировка вектора ограничений сверху на
  оптимизационные переменные (3.15);

- большая часть остальных данных копируется из блока для задачи на
  множестве $D_{i}^{S'}$;

- $A^{k}$, $m$, $n^{k}$ - одинаковы для всех задач.

При формировании данных по задаче $D_{i2}^{S'}$:

- осуществляется корректировка вектора ограничений сверху на
  оптимизационные переменные (3.17);

- осуществляется корректировка вектора правых частей ограничений (3.16);

- осуществляется корректировка вектора
  $\overline{b}(B_{\text{opt}},N_{\text{opt}}^{})$:
  $\overline{b}(B_{\text{opt}},N_{\text{opt}}^{}) = {\overline{B}}^{- 1}b_{D_{2}^{\text{si}}}$,
  где
  ${\overline{B}}^{- 1}b_{D_{2}^{\text{si}}} = \left\lbrack \frac{b_{(D_{i2})}}{0} \right\rbrack$.

- большая часть остальных данных (в том числе и ${\overline{B}}^{- 1}$)

- $A^{k}$, $m^{k}$, $n^{k}$ - одинаковы для всех задач.

## Реализация метода правильных отсечений

Исходная постановка оптимизационной задачи имеет вид:

$\underset{x^{n}}{\text{max}c^{n}x^{n}}\text{}D^{n} =$. (3.20)

Предполагается, что матрицы и вектора $A^{n}$, $b^{n}$ и $h^{n}$
($\forall h^{n}\lbrack j\rbrack < \infty$) заданы в целочисленном виде.

Укрупненная структура алгоритма МПО ничем не отличается от описанной в
\[53\].

Ниже излагаются особенности детализации отдельных процедур укрупненного
алгоритма, связанные с применением его к задаче с двухсторонними
ограничениями на оптимизационные переменные.

### Особенности алгоритмической реализации метода правильных отсечений

При решении ЗЛП, полученных из исходной или расширенной задачи путем
снятия условий целочисленности, применяются специальные алгоритмы
линейного программирования, алгоритмически учитывающие двусторонние
ограничения на оптимизационные переменные:

На первом этапе работы алгоритма МПО (при решении исходной задачи (3.1)
со снятыми условиями целочисленности) применяется прямой специальный
алгоритм решения ЗЛП с двухсторонними ограничениями на оптимизационные
переменные;

На больших итерациях алгоритма МПО применяется специальный алгоритм
двойственного симплекс-метода, детали которого изложены в разделе
3.3.1.2.

В результате выполнения первого этапа ЗЛП, полученная из (3.20),
приводится к канонической форме:

$\underset{x^{k}}{\text{max}c^{k}x^{k}}\text{}D^{k} =$, (3.21)

где:

- $n_{k} = n_{n} + n_{\delta}$, $n_{\delta}$ - число балансовых
  переменных, вводимых в задачу при переходе от неравенств к равенствам;

- Вектор $b^{k}$ отличается в общем случае от $b^{n}$ тем, что все его
  компоненты неотрицательны. Это преобразование делается при
  формировании вспомогательной задачи перед поиском допустимого
  базисного решения;

- $A^{k}$ отличается от $A^{n}$ за счет введения балансовых переменных и
  преобразования ограничений к виду, когда их правые части будут
  неотрицательными.

- $h^{k}$ получается путем добавления к $h^{n}$ $n^{\delta}$ компонент,
  определенных как $\infty$; $c^{k}$ получается из $c^{n}$ путем
  добавления $n^{\delta}$ нулевых компонент.

Число ограничений в задачах (3.20) и (3.21) совпадает, обозначим его
$m$.

Специальный алгоритм двойственного симплекс метода работает с
расширенной задачей, которая отличается от задачи (3.21) за счет
введения ограничений, называемых правильными отсечениями.

### Описание процедуры сжатия.

Процедура сжатия осуществляется после завершения очередной большой
итерации.

#### Исходные данные

$n_{k}$ - число переменных в задаче (3.21).

$n_{R}$ - число переменных в задаче (3.22), полученной к данному этапу
работы алгоритма Гомори.

$m$ - число ограничений в задаче (3.21).

$m_{R}$ ~-~ число ограничений в расширенной задаче (3.22);

${\overline{A}}^{R}$ - Расширенная матрица условий расширенной задачи
(3.22).

$b^{R}$ - вектор правых частей ограничений расширенной задачи (3.22).

$T^{R}(B,N^{})$ - заключительная симплекс-таблица расширенной задачи
(3.22), полученная после завершения очередной большой итерации.

Скорректированные данные по расширенной задаче после сжатия будем
отмечать знаком "\~".

#### Алгоритм сжатия

1.  Просмотреть базисное множество $N_{R}(B)$ на предмет наличия в нем
    переменных, номера которых больше $n_{k}$~.~ Если таких переменных
    нет, то работа процедуры сжатия завершена. Переход на 5. Иначе
    определен номер первой встретившейся в $N_{R}(B)$ переменной
    $j > n_{k}$ и порядковый номер $1$, определяющий положение этой
    переменной в базисе.

2.  Вычислить номер ограничения в расширенной задаче (правильного
    отсечения), для которого $j$ является балансовой переменной:
    $k = m + (j - n_{k})$

3.  Во всех ограничениях с номерами $i > k$ исключить $j$-ю переменную
    (корректируются строки $a_{i}^{R}$ матрицы $A^{R}$ и элементы
    вектора $b^{R}$):\
    $\begin{array}{r}
    \left. \ a_{i}^{R} = a_{i}^{R} - a\lbrack i,j\rbrack a_{k}^{R} \middle| \right\}
    \end{array}\text{}i = k + 1,\text{...},m_{R}$

4.  Произвести сжатие: ${\overset{\sim}{m}}_{R} = m_{R} - 1$,
    ${\overset{\sim}{n}}_{R} = n_{R} - 1$

    1.  Из ${\overline{A}}^{R}$ удалить $k$-ю строку
        (${\overline{a}}_{k}^{R}$) и $j$-й столбец
        (${\overline{a}}_{k}^{j}$).

    2.  Из $b^{R}$ удалить $k$-й элемент ($b^{R}\lbrack k\rbrack$).

    3.  Из базисного множества удалить переменную $j$ ($l$-й элемент), а
        номера всех переменных, которые больше $j$, уменьшить на
        единицу.

    4.  Из $\overline{b}(B,N \ast )$ удалить $l$-й элемент
        ($\overline{b}(B,N \ast )\lbrack l\rbrack$)

    5.  Из ${\overline{B}}^{- 1}$ удалить $l$-ю строку
        ${\overline{d}}_{l}$ и $k$-й столбец ${\overline{d}}^{k}$.

5.  Конец.

### Оценка накапливаемой при осуществлении ПГЖ погрешности и использование ее для определения идентификации целочисленных базисных компонент

В результате итеративных ПГЖ накапливаются погрешности, которые
затрудняют идентификацию целочисленности базисных компонент после
проведения каждой большой итерации алгоритма МПО. Это может привести к
сбоям в работе алгоритма при проверке условий окончания его работы и
выборе производящей строки, если условия не выполняются.

В данной работе излагается процедура оценки накопленной погрешности и
использование ее для выявления целочисленных значений базисных
компонент. Алгоритм основан на расчете фактических значений матрицы,
которая должна быть единичной. При расчете этой матрицы используется
обратная базисная матрица ЗЛП, элементы которой посчитаны с фактической
погрешностью, накопленной в результате итеративных преобразований.

По результатам сравнения соответствующих элементов рассчитанной матрицы
и идеальной (единичной) матрицы определяется максимальная накопленная
погрешность вычислений, которая затем используется для идентификации
целочисленности базисных компонент.

Процедура вычисления накопленной погрешности осуществляется с некоторым
шагом по числу больших итераций алгоритма МПО.

#### Алгоритм расчета накопленной погрешности

Алгоритм расчета накопленной погрешности имеет следующий вид:

Формирование в соответствии с базисным множеством $N_{R}(B)$ матрицы
$\overline{B}$ (в соответствии с $N_{R}(B)$ из матрицы
${\overline{A}}^{R}$ берутся столбцы и переносятся в матрицу
$\overline{B}$. Таких столбцов $m_{R}$ Последний столбец определяется
как $e_{m_{R} + 1}$).

Определяется произведение
${\overline{E}}' = \overline{B} \times {\overline{B}}^{- 1}$, где
${\overline{B}}^{- 1}$ - расширенная обратная матрица полученная на
основе ПГЖ, а матрица $B$ формируется по базисному множеству на основе
исходных данных решаемой ЗЛП.

Определяется матрица погрешностей:
${\overline{E}}^{\Delta} = {\overline{E}}' - \overline{E}$, где
$\overline{E}$ - единичная матрица размерности
$(m_{R} + 1) \times (m_{R} + 1)$.

Определяется максимальное значение абсолютной погрешности:

$$\varepsilon_{\text{max}} = \underset{i,j \in (1\text{...}m_{R} + 1)}{\text{max}}\left| {\overline{e}}^{\Delta}\lbrack i,j\rbrack \right|$$

Идентификация целочисленности базисных компонент следущим образом:

Компоненты вектора $b(B,N^{})$ округляются до ближайшего целого значения
$b'(B,N^{})$.

Компоненты, для которых
$b'(B,N^{})\lbrack i\rbrack - b(B,N^{})\lbrack i\rbrack > \varepsilon_{\text{max}}$,
считаются нецелочисленными.

## Сравнение альтернативных методов поиска оптимального решения математической модели линейного программирования

### Критерии сравнения

Одной из основных проблем математического программирования является
высокая вычислительная сложность методов оптимизации. Особенно это
относится к целочисленному программированию, существующие методы
которого имеют экспоненциально возрастающую вычислительную сложность при
увеличении размерности задачи по числу оптимизационных переменных. В
связи с этим в качестве основного показателя критерия сравнения
выбирается машинное время нахождения оптимального решения. Для его
фиксации в разработанные программы включены соответствующие средства.

Конечный результат оптимизации у каждого из сравниваемых методов
потенциально один и тот же -- оптимальное решение задачи (если, конечно,
оно существует; возможным исходом является пустота области допустимых
решений). Однако, как уже говорилось выше, некоторые из сравниваемых
методов имеют недостатки, определяемые погрешностями машинных
вычислений. Это прежде всего касается МПО и в значительно меньшей
степени МВГ. Накопление погрешности может привести к пропуску
оптимального решения, а в худшем случае -- к аварийному сбою в работе
алгоритма оптимизации. Поэтому в качестве следующего показателя
сравнения выберем надежность поиска оптимального решения, которую можно
оценить по частоте сбоя программы из-за накопленной погрешности и
частоту пропуска оптимального решения. Эти частоты могут быть объективно
определены на множестве решаемых задач, так как среди сравниваемых есть
методы (МНПВР, КМ), принципиально свободные от указанных недостатков.

Все сравниваемые методы потенциально могут находить точное оптимальное
решение. Но иногда из-за временных ограничений или других соображений у
пользователя может возникнуть потребность поиска хотя бы какого либо
допустимого решения (лучше субоптимального), но как можно быстрее.
Поэтому свойство алгоритма оптимизации давать допустимое решение
оптимизационной задачи до окончания полного ее решения да еще, если это
возможно, с оценкой его точности является важным показателем метода
\[54\].

### Генератор линейных задач целочисленного программирования

Цели исследования требуют генерации большого числа разнообразных задач
возрастающей размерности по числу оптимизационных переменных и
ограничений, размерам гиперпараллепипеда полного перебора, имеющих
различную конфигурацию области допустимых решений и направление по
отношению к ней градиента целевой функции. Формирование множества таких
задач практически невозможно без автоматизации. Для этого создан
программный генератор \[20\], который работает следующим образом.

Область допустимых решений формируется в положительном квадранте
$n$-мерного пространства по числу оптимизационных переменных внутри
гиперпараллепипеда полного перебора. Исходными параметрами для
генератора помимо числа переменных $n$ являются также число ограничений
$m$, величины ограничений на оптимизационные переменные сверху
$h\lbrack j\rbrack\text{  }(j = 1,n)$ и заданная вероятность
формирования ограничений, тип которых отличен от ресурсных.

Ограничения определяются в виде неравенств типа "$$" с положительными
правыми частями. Формирование ограничений осуществляется путем генерации
псевдослучайных отношений правой части ограничения к коэффициентам при
каждой переменной. Затем по сгенерированным отношениям в соответствии с
определенными в программе значениями правых частей вычисляются и сами
коэффициенты ограничений. Ограничения могут формироваться двух типов:
наиболее распространенные, так называемые, ресурсные, у которых все
коэффициенты в левой части являются неотрицательными числами;
ограничения не ресурсного типа, в которых при отдельных оптимизационных
переменных коэффициенты могут быть отрицательными числами. Тип
ограничения формируется стохастически по заданной вероятности включения
в формируемое ограничение отрицательных коэффициентов при переменных.

Коэффициенты целевой функции генерируются как независимые равномерно
распределенные на отрезке $\lbrack 0 - \text{100}\rbrack$
псевдослучайные величины.

Важной характеристикой сформированной задачи, влияющей на сравнительный
анализ различных методов оптимизации (особенно, методов неявного
перебора) является так называемая «жесткость» ограничений, определяющая
величину той части гиперпараллепипеда полного перебора, которая занята
областью допустимых решений. Для оценки сформированной задачи по этому
показателю в генератор включен имитатор $n$-мерного дискретного
равномерного распределения в гиперпараллепипеде полного перебора, с
помощью которого по методу Монте-Карло оценивается указанный показатель.

#### Сравнение по надежности поиска оптимального решения.

Временные рамки проведения машинного эксперимента не позволили в полной
мере провести сравнение по этому показателю. Априорный анализ алгоритмов
показывает, что надежность МНПВР и КМ вне подозрений. Дело в том, что
переборные методы, к которым относится и МНПВР, вообще не подвержены
ошибкам, связанным с накоплением погрешностей вычислений, так как в них
отсутствуют процедуры работы с действительными числами. КМ на основных
этапах поиска решения (втором и четвертом) использует именно МНПВР, а
возможные ошибки вычислений на первом и третьем этапах при работе
алгоритма симплекс-таблиц не могут принципиально сказаться на
результатах поиска: на этих этапах определяются лишь границы поиска для
следующих за ними этапов КМ и ошибки, в крайнем случае, могут привести
лишь к некоторому расширению области поиска и, соответственно,
увеличению времени решения задачи.

В то же время, использование алгоритмов симплекс-таблиц в МПО и МВГ
принципиально отличается от КМ тем, что там эти алгоритмы применяются
непосредственно для нахождения оптимального решения. Накапливаемые при
многократном применении алгоритмов симплекс-таблиц погрешности при
выполнении процедур выявления целочисленных компонент решения, выделения
дробных частей действительных чисел могут привести к принципиальным
ошибкам: пропуску оптимального решения или вообще к аварийному сбою в
работе алгоритма. В этом смысле МВГ обладает некоторыми преимуществами
по сравнению с МПО из-за простоты вводимых новых ограничений при
ветвлении. Для повышения надежности МПО, как уже говорилось выше, в него
включены специальные процедуры оценки накопленной погрешности вычислений
и использования ее для уменьшения вероятности возможных ошибок.

Приведенный анализ ограниченно подтвердился при проведении машинных
экспериментов: в двух задачах из множества, на которых тестировался и
исследовался МПО произошел пропуск оптимального решения с нахождением
решения, более худшего по сравнению с оптимальным по значению целевой
функции.

#### Сравнение по возможности досрочного результативного завершения работы алгоритма.

По этому показателю наибольшими преимуществами по сравнению с другими
методами обладают МВГ и КМ, в которых для найденных на ранних этапах
допустимых решений очень просто получаются оценки сверху возможного их
отклонения от оптимального по значению целевой функции близкие к
допустимому решению. Затем следует МНПВР, где допустимое решение может
быть получено задолго до окончания работы алгоритма, но без оценки его
отклонения от оптимального. Наихудшим по рассматриваемому показателю
является МПО, в котором до окончательного завершения работы алгоритма не
определяется ни одного допустимого решения исходной задачи.

# Решение задачи определения оптимальной топологии ЛВС комбинированным методом

## ЛВС с применением концентраторов разных типов и непосредственным подключением пользователей к центральному концентратору

### Преобразования задачи с целью уменьшения ее размерности и эффективного применения комбинированного метода.

Применение комбинированного метода требует перехода к одноиндексным
переменным. Именно с такими задачами работают алгоритмы симплекс-таблиц
и метод неявного перебора по векторной решетке, являющиеся составными
частями комбинированного метода. Для осуществления такого преобразования
необходимо каждой переменной $x\lbrack i,j\rbrack$ поставить в
соответствие одноиндексную переменную. Перенумерация возможна в
естественном порядке (по строкам матрицы $x\lbrack i,j\rbrack$). При
осуществлении такой пере нумерации целесообразно одновременно исключить
из множества оптимизационных переменных те, для которых
$d\lbrack i,j\rbrack > D_{\text{max}}$, уменьшив тем самым размерность
задачи по числу переменных.

Из числа оптимизационных переменных также целесообразно исключить
переменные $x\lbrack i,j\rbrack$ с индексами $j$, соответствующими тем
помещениям, где отсутствуют пользователи. По этой же причине
одновременно необходимо из числа ограничений второго типа исключить те,
которые определяют выделение линий связи для пользователей в этих
помещениях.

Аналогичную перенумерацию необходимо сделать и по переменным
$y\lbrack i,j\rbrack$.

Число оптимизационных переменных, соответствующих $x\lbrack i,j\rbrack$
и $y\lbrack i,j\rbrack$, может быть также уменьшено, если есть
помещения, в которых либо нельзя, либо, исходя из предварительного
анализа матрицы $d\lbrack i,j\rbrack$, нецелесообразно размещать
концентраторы.

Целесообразно еще сделать преобразование ограничений второго типа к виду
неравенства. Дело в том, что МНПВР, являющийся составной частью КМ,
работает лишь с ограничениями типа неравенств. Ограничения второго типа
представляют собой по исходной постановке равенства. Если не сделать
предварительных преобразований, то сам алгоритм оптимизации
автоматически преобразует их в неравенства, поставив в соответствие
каждому из равенств два неравенства противоположных знаков. Очевидно,
что такое преобразование приведет к увеличению числа ограничений и, как
следствие, к определенному увеличению вычислительной сложности процесса
оптимизации. Можно избежать этого, заменив указанные равенства с самого
начала на неравенства типа "$$":

$$\sum_{i = 1}^{n}{x\lbrack i,j\rbrack \geq b\lbrack j\rbrack\text{   }(j = 1,n)}$$

С учетом знака неравенства ограничений первого типа и направления
оптимизации на минимум такое изменение приведет к эквивалентной по
отношению к исходной оптимизационной задаче без последующего увеличения
числа ограничений.

Заключительным этапом преобразований являются следующие формальные
преобразования ограничений:

- все члены с оптимизационными переменными переносятся в левую часть
  ограничений, а константы -- в правую часть;

- все ограничения приводятся к виду, где правые части являются
  неотрицательными числами. Это необходимо для осуществления начального
  этапа симплекс-метода -- поиска допустимого решения методом
  минимизации невязок;

- ограничения четвертого типа записываются отдельно для левого и правого
  условий;

- вектор ограничений $h$ на оптимизационные переменные сверху
  определяется следующим образом:

  - для каждой переменной $$$x\lbrack i,j\rbrack$ соответствующее
    $h\lbrack l\rbrack = b\lbrack j\rbrack$;

  - значение $h\lbrack l\rbrack$, соответствующее $y\lbrack 1,i\rbrack$,
    рассчитывается по формуле
    $h\lbrack l\rbrack = \lbrack(b\lbrack j\rbrack + 1\frac{)}{a}\rbrack + 1$;

  - значение $h\lbrack l\rbrack$, соответствующее $y\lbrack 2,i\rbrack$,
    рассчитывается по формуле
    $h\lbrack l\rbrack = \lbrack b\lbrack j\frac{\rbrack}{(}a - 1)\rbrack + 1$;

  - значение $h\lbrack l\rbrack$ для любой $w\lbrack i\rbrack$ равно
    единице.

### Эвристический алгоритм формирования допустимого решения задачи

Условия задачи позволяют сформулировать некоторые правила, на основе
которых удалось значить улучшить свойства комбинированного в применении
к этой задаче. Приведенный ниже эвристический алгоритм предлагается
использовать вместо линейного программирования на первом этапе
комбинированного метода.

1.  Для каждого $i$--го помещения ($i = \overline{1,n}$) рассчитывается
    суммарное число еще не подключенных пользователей по помещениям,
    непосредственно связанным с $i$‑м:
    $\text{КПS}\lbrack i\rbrack = \sum_{j \in I_{i}}^{}{\text{КП}\lbrack j\rbrack}$,
    где $I_{i}$ - множество помещений, имеющих непосредственную связь с
    $i$‑м (включая само $i$‑е).

2.  Определяется возможность установки в $i$‑м узле стека из
    концентраторов первого тип, если возможно, то и их количество.

    1.  Если $\text{КПS}\lbrack i\rbrack > k \cdot a - 1$, где
        $k \simeq 1\text{.}5 \div 2$, и оставшееся для распределения
        количество концентраторов первого типа больше двух, то можно
        устанавливать в $i$‑м узле стек из концентраторов первого типа в
        количестве:
        $\text{КК}1\lbrack i\rbrack = \text{min}\left\{ \left\lbrack \frac{\text{КПS}\lbrack i\rbrack + 1}{a} \right\rbrack,k_{\text{max}} \right\}$,
        при этом количество портов для подключения пользователей на
        данном шаге алгоритма рассчитывается как
        $\text{КПП}\lbrack i\rbrack = a \cdot \text{КК}1\lbrack i\rbrack - 1$;

    2.  Иначе рассматривается возможность установки в $i$‑м узле одного
        концентратора второго типа
        ($\text{КПП}\lbrack i\rbrack = a - 1$).

3.  Рассчитывается сумма длин соединений, приходящихся на одного
    подключаемого пользователя к концентратору(ам), устанавливаемого(ых)
    в $i$‑м узле:
    $\text{SD}\lbrack i\rbrack = \frac{d\lbrack i,k\rbrack + \sum_{j \in I_{i}}^{}{КП'\lbrack j\rbrack d\lbrack i,j\rbrack}}{{\text{КП \{}S}'}\lbrack i\rbrack$,
    где $КП'\lbrack j\rbrack$ - количество пользователей $j$‑го
    помещения, подключаемых к установленному(ым) в $i$‑м узле
    концентратору(рам) (выбор $j$‑х узлов, из которых в первую очередь
    идет подключение, осуществляется по минимальному расстоянию
    $d\lbrack i,j\rbrack$). Подключение пользователей (определение
    $КП'\lbrack j\rbrack$) идет до тех пор, пока либо будут использованы
    все порты планируемого(ых) для установки концентраторов, либо будут
    подключены все пользователи:
    ${\text{КП \{}S}'\lbrack i\rbrack = \sum_{j \in I_{i}}^{}{КП'\lbrack j\rbrack}$.

4.  Производится ранжирование помещений по величине
    $\text{КПS}\lbrack i\rbrack$ (чем больше, тем больше ранг
    $R1\lbrack i\rbrack$). При одинаковом $\text{КПS}\lbrack i\rbrack$
    ранг определяется по величине $\text{КП}\lbrack i\rbrack$.

5.  Производится ранжирование помещений по величине
    $\text{SD}\lbrack i\rbrack$ (чем меньше, тем больше ранг
    $R2\lbrack i\rbrack$.

6.  Определяется суммарный ранг:
    $\text{RS}\lbrack i\rbrack = R1\lbrack i\rbrack + R2\lbrack i\rbrack$.

7.  Определяется помещение с максимальным рангом:
    $\underset{i}{\text{max}}\text{RS}\lbrack i\rbrack$.

8.  Реализовать для выбранного помещения размещение концентратора(ов) и
    подключение пользователей в количестве $КП'\lbrack j\rbrack$.

9.  Пересчитать количество оставшихся в каждом помещении неподключенных
    пользователей и количество оставшихся концентраторов.

10. Если все пользователи подключены, то конец работы алгоритма.

11. Остались ли неиспользованные концентраторы? Если да, то на п.1,
    иначе допустимое решение не найдено.

12. Конец

### Преобразование задачи для применения комбинированного метода

Применение комбинированного метода требует перехода к единым
одноиндексным переменным. Именно с такими задачами работают алгоритмы
симплекс-таблиц и метод неявного перебора по векторной решетке,
являющиеся составными частями комбинированного метода. Из числа
оптимизационных переменных также имеет смысл исключить переменные
$y\lbrack v\rbrack$ с индексами $v$, соответствующие тем комнатам, где
нет пользователей. По этой же причине необходимо из числа ограничений
(4.8), (4.10) и (4.11) исключить те, которые определяют выделение
концентраторов для пользователей в этих комнатах.

Число оптимизационных переменных, соответствующих $y\lbrack v\rbrack$,
может быть также уменьшено, если есть помещения, в которых либо нельзя,
либо, исходя из предварительного анализа матрицы
$c\lbrack\text{ij}\rbrack$, нецелесообразно размещать концентраторы.

Целесообразно еще сделать преобразование первого типа к виду
неравенства. Дело в том, что МНПВР, являющийся составной частью КМ,
работает лишь с ограничениями типа неравенств. Ограничения второго типа
представляют собой по исходной постановке равенства.

Заключительным этапом преобразований являются следующие формальные
преобразования ограничений:

- все члены с оптимизационными переменными переносятся в левую часть
  ограничений, а константы -- в правую часть;

- все ограничения приводятся к виду, где правые части являются
  неотрицательными числами. Это необходимо для осуществления начального
  этапа симплекс-метода -- поиска допустимого решения методом
  минимизации невязок;

- вектор ограничений $h$ на оптимизационные переменные сверху
  определяется следующим образом:

  - для каждой переменной $x\lbrack d\rbrack$ соответствующее значение
    $h\lbrack l\rbrack = \text{KPS}$;

  - для каждой переменной $y\lbrack v\rbrack$ соответствующее значение
    $h\lbrack l\rbrack \leq \left\{ \left\lbrack \frac{\text{KP}\lbrack v\rbrack + (n - 1) - 1}{a - 2} \right\rbrack + 1 \right\}$

  - для каждой переменной $z\lbrack d\rbrack$ соответствующее значение
    $h\lbrack l\rbrack = 1$
